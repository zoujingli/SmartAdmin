# 插件实战教程：从零开发 Todo 插件

本教程用一个最小但完整的 `Todo` 后台插件演示 SmartAdmin 插件机制。跟着本页复制文件后，你应能得到一个具备本地 Composer path 包、Provider 装配、`plugin.json` 菜单权限、迁移、后端 CRUD、插件前端页面、菜单节点同步、构建和 ZIP 打包能力的插件。

> 本页是“照着做”的实战教程；机制细节、字段含义和边界请配合阅读 [插件开发](./插件开发.md)。

## 目标效果

| 项目 | 本教程固定值 |
|------|--------------|
| 插件目录 | `plugin/Todo` |
| Composer 包 | `zoujingli/smart-plugin-todo` |
| PHP 命名空间 | `Plugin\Todo` |
| 表前缀 | `todo_` |
| 示例表 | `todo_task` |
| 后台接口 | `/system/todo/task/**` |
| 后台菜单 | `/system/todo/task` |
| 权限前缀 | `todo.task.*` |
| 前端根 | `plugin/Todo/stc/view` |

Todo 示例字段固定为：

```text
id, tenant_id, title, content, priority, status, done_at, sort, remark,
created_by, updated_by, created_at, updated_at, deleted_at
```

字段语义：`status=0` 表示待办，`status=1` 表示完成；完成时写入 `done_at`，重新改回待办时清空 `done_at`。

## 1. 创建插件目录

在项目根目录创建目录结构：

```bash
mkdir -p plugin/Todo/src/{Controller,Mapper,Model,Service}
mkdir -p plugin/Todo/stc/{migrations,view/task}
```

最终结构应类似：

```text
plugin/Todo/
├── composer.json
├── plugin.json
├── readme.md
├── src/
│   ├── Controller/SystemTaskController.php
│   ├── Mapper/TodoTaskMapper.php
│   ├── Model/TodoTask.php
│   ├── Provider.php
│   └── Service/TodoTaskService.php
└── stc/
    ├── migrations/2026_05_29_000001_todo_task.php
    └── view/task/index.vue
```

> `plugin/Todo` 是示例名称。开发真实插件时，将 `Todo`、`todo`、`todo_task`、`Plugin\Todo`、`zoujingli/smart-plugin-todo` 统一替换成你的业务名称。

## 2. 接入根 composer.json

根 `composer.json` 负责让本地插件包参与 Composer autoload 和 Hyperf Provider 合并。新增插件时，加入 path repository 和 require。

示例片段：

```json
{
  "require": {
    "zoujingli/smart-plugin-todo": "^1.0"
  },
  "repositories": [
    {
      "type": "path",
      "url": "plugin/Todo"
    }
  ]
}
```

然后执行：

```bash
composer update zoujingli/smart-plugin-todo
```

注意：

- Composer 只负责本地 path 包、autoload 和 Provider 装配。
- 菜单、权限、view、语言包、迁移目录不写入 Composer 元数据，统一写入 `plugin.json`。
- 修改 Composer、Provider 或 autoload 后，需要重启 Hyperf/Swoole 进程。

## 3. 创建插件 composer.json

创建 `plugin/Todo/composer.json`：

```json
{
  "name": "zoujingli/smart-plugin-todo",
  "description": "SmartAdmin Todo plugin demo",
  "type": "library",
  "license": "Apache-2.0",
  "autoload": {
    "psr-4": {
      "Plugin\\Todo\\": "src/"
    }
  },
  "extra": {
    "hyperf": {
      "config": "Plugin\\Todo\\Provider"
    }
  }
}
```

关键点：

- `name` 必须与根 `composer.json require` 一致。
- `autoload.psr-4` 必须指向插件 `src/`。
- `extra.hyperf.config` 必须指向插件 `Provider`。
- 插件授权由 `license` 和 `plugin.json.plugin.license_*` 共同表达。

## 4. 创建 Provider

创建 `plugin/Todo/src/Provider.php`：

```php
<?php

declare(strict_types=1);

namespace Plugin\Todo;

final class Provider
{
    public function __invoke(): array
    {
        return [
            'annotations' => [
                'scan' => [
                    'paths' => [__DIR__],
                ],
            ],
        ];
    }
}
```

Provider 边界必须清楚：

- 只负责运行期装配，例如注解扫描、依赖绑定、监听器、命令。
- 不注册菜单，不声明模块，不写 view 路径。
- 不保存请求态、用户态、租户态或 Token 上下文。

## 8. 创建 Mapper

创建 `plugin/Todo/src/Mapper/TodoTaskMapper.php`：

```php
<?php

declare(strict_types=1);

namespace Plugin\Todo\Mapper;

use Hyperf\Database\Model\Builder;
use Library\CoreMapper;
use Plugin\Todo\Model\TodoTask;

final class TodoTaskMapper extends CoreMapper
{
    public function __construct(
        protected string $model = TodoTask::class
    ) {}

    /**
     * Todo 列表只允许白名单筛选，不能让请求直接控制 select、orderBy 或 raw SQL。
     */
    protected function handleSearch(Builder $query, array $params): Builder
    {
        $builder = _query($query, $params)
            ->equal('status,priority')
            ->like('title,content,remark')
            ->dateBetween('created_at')
            ->getQuery();

        $keyword = trim((string)($params['keyword'] ?? ''));
        if ($keyword !== '') {
            $builder->where(function (Builder $subQuery) use ($keyword): void {
                $like = "%{$keyword}%";
                $subQuery->where('title', 'like', $like)
                    ->orWhere('content', 'like', $like)
                    ->orWhere('remark', 'like', $like);
            });
        }

        return $builder->orderBy('sort', 'desc')->orderBy('id', 'desc');
    }

    /**
     * 列表额外统计给前端卡片使用，仍走当前租户和筛选边界。
     */
    protected function handleListExtra(array $params = [], bool $isScope = true): array
    {
        return $this->buildStatusListExtra($params, $isScope);
    }
}
```

Mapper 只负责查询和数据访问，不读取 HTTP Request，也不承载业务校验。

## 9. 创建 Service

创建 `plugin/Todo/src/Service/TodoTaskService.php`：

```php
<?php

declare(strict_types=1);

namespace Plugin\Todo\Service;

use Carbon\Carbon;
use Library\CoreService;
use Plugin\Todo\Mapper\TodoTaskMapper;

final class TodoTaskService extends CoreService
{
    public function __construct(
        protected TodoTaskMapper $mapper
    ) {}

    /**
     * 状态流转由 Service 统一处理，保证完成状态和 done_at 同步变化。
     */
    public function changeStatus(int $id, int $status): bool
    {
        return $this->update($id, ['status' => $status]);
    }

    /**
     * 写入数据只允许白名单字段，复杂默认值和状态联动在这里完成。
     */
    protected function filterData(array &$data, array $exists = []): array
    {
        $rules = [
            'title.max:100' => '任务标题最多 100 个字符',
            'content.max:2000' => '任务内容最多 2000 个字符',
            'priority.integer' => '优先级必须为数字',
            'priority.in:0,1,2' => '优先级取值错误',
            'status.integer' => '任务状态必须为数字',
            'status.in:0,1' => '任务状态取值错误',
            'sort.integer' => '排序值必须为数字',
            'remark.max:255' => '备注最多 255 个字符',
        ];
        if ($exists === []) {
            $rules['title.required'] = '任务标题不能为空';
            $rules['priority.default'] = 0;
            $rules['status.default'] = 0;
            $rules['sort.default'] = 0;
        }

        $data = _vali($rules, $data);
        foreach (['priority', 'status', 'sort'] as $field) {
            if (array_key_exists($field, $data)) {
                $data[$field] = (int)$data[$field];
            }
        }

        if (array_key_exists('status', $data)) {
            $data['done_at'] = (int)$data['status'] === 1 ? Carbon::now()->toDateTimeString() : null;
        }

        return $data;
    }
}
```

Controller 不做字段校验；`filterData()` 是写入验证和过滤的标准入口。

## 10. 创建 Controller

创建 `plugin/Todo/src/Controller/SystemTaskController.php`：

```php
<?php

declare(strict_types=1);

namespace Plugin\Todo\Controller;

use Hyperf\HttpServer\Annotation\Controller;
use Hyperf\HttpServer\Annotation\DeleteMapping;
use Hyperf\HttpServer\Annotation\GetMapping;
use Hyperf\HttpServer\Annotation\PostMapping;
use Hyperf\HttpServer\Annotation\PutMapping;
use Hyperf\HttpServer\Contract\RequestInterface;
use Library\CoreController;
use Library\Events\Annotation\Auth;
use Library\Events\Annotation\Logger;
use Plugin\Todo\Service\TodoTaskService;

#[Auth(name: 'Todo任务管理')]
#[Controller(prefix: 'system/todo/task')]
final class SystemTaskController extends CoreController
{
    public function __construct(
        protected TodoTaskService $service
    ) {}

    #[GetMapping(path: 'index')]
    #[Auth(name: 'Todo任务列表', type: Auth::CHECK, menu: true, code: 'todo.task.index')]
    public function index(RequestInterface $request): array
    {
        $this->success('获取成功', $this->service->getPageList($request->all()));
    }

    #[GetMapping(path: 'recycle')]
    #[Auth(name: 'Todo任务回收站', type: Auth::CHECK, menu: false, code: 'todo.task.index')]
    public function recycle(RequestInterface $request): array
    {
        $this->success('获取成功', $this->service->getRecycleList($request->all()));
    }

    #[GetMapping(path: 'info/{id}')]
    #[Auth(name: 'Todo任务详情', type: Auth::CHECK, menu: false, code: 'todo.task.index')]
    public function info(int $id): array
    {
        $this->respondFound($this->service->read($id));
    }

    #[PostMapping(path: 'create')]
    #[Auth(name: '新增Todo任务', type: Auth::CHECK, menu: false, code: 'todo.task.create')]
    #[Logger(name: '新增Todo任务')]
    public function create(RequestInterface $request): array
    {
        $this->success('创建成功', $this->service->create($request->all()));
    }

    #[PutMapping(path: 'update/{id}')]
    #[Auth(name: '编辑Todo任务', type: Auth::CHECK, menu: false, code: 'todo.task.update')]
    #[Logger(name: '编辑Todo任务')]
    public function update(int $id, RequestInterface $request): array
    {
        $this->service->update($id, $request->all());
        $this->success('更新成功', $this->service->read($id));
    }

    #[PutMapping(path: 'status/{id}')]
    #[Auth(name: '更新Todo任务状态', type: Auth::CHECK, menu: false, code: 'todo.task.status')]
    #[Logger(name: '更新Todo任务状态')]
    public function status(int $id, RequestInterface $request): array
    {
        $this->service->changeStatus($id, (int)$request->input('status', 0));
        $this->success('更新成功', $this->service->read($id));
    }

    #[DeleteMapping(path: 'delete/{ids}')]
    #[Auth(name: '删除Todo任务', type: Auth::CHECK, menu: false, code: 'todo.task.delete')]
    #[Logger(name: '删除Todo任务')]
    public function delete(string $ids): array
    {
        $this->deleteByIds($ids, fn (array $idArray) => $this->service->delete($idArray));
    }

    #[PutMapping(path: 'recovery/{ids}')]
    #[Auth(name: '恢复Todo任务', type: Auth::CHECK, menu: false, code: 'todo.task.recovery')]
    #[Logger(name: '恢复Todo任务')]
    public function recovery(string $ids): array
    {
        $idArray = $this->idsOrFail($ids);
        $this->service->recovery($idArray);
        $this->success('恢复成功', $idArray);
    }

    #[DeleteMapping(path: 'real-delete/{ids}')]
    #[Auth(name: '彻底删除Todo任务', type: Auth::CHECK, menu: false, code: 'todo.task.real-delete')]
    #[Logger(name: '彻底删除Todo任务')]
    public function realDelete(string $ids): array
    {
        $this->deleteByIds($ids, fn (array $idArray) => $this->service->delreal($idArray), '彻底删除成功');
    }
}
```

Controller 要点：

- 类必须 `final class ... extends CoreController`。
- 受保护接口必须加 `#[Auth]`，权限码和 `plugin.json` 保持一致。
- 写操作必须加 `#[Logger]`；有敏感字段时用 `excludeFields` 排除。
- 响应统一用 `$this->success()`、`$this->respondFound()`、`$this->deleteByIds()`。

## 12. 同步菜单和权限节点


如果菜单出现但接口返回 403，优先检查：

1. `plugin.json permissions[].code` 是否与 Controller `#[Auth(code: ...)]` 一致。
2. 是否执行过 `xadmin:node:sync`。
3. 当前角色是否分配了对应权限。
4. 浏览器是否仍缓存旧菜单和旧权限码。

## 13. 构建前端

插件页面、`routes.ts`、`auth-entry.ts` 或 API service 变更后必须重新构建前端：

```bash

```

开发时如 Vite dev 已启动，新增 `plugin.view_root` 或新增插件目录后建议重启 dev 服务；只改已有 Vue 文件通常热更新即可。

发布包规则：

- `composer web:build` 负责生成前端产物。
- `composer build` 只复用已有 `web/dist` 并打入发布包。
- 不能指望生产 Phar/SFX 在运行时动态扫描新 Vue 页面。

`xadmin:plugin:*` 命令只在源码/CI 模式可用，命令类必须走 `SourceOnlyCommand`；生产 Phar/SFX 不注册这些命令。

打包 Todo 插件：

安装 ZIP 插件：

```bash

composer web:build
./bin/smart.php xadmin:menu:sync --dry-run --json
./bin/smart.php xadmin:node:sync --dry-run --json
```

备份插件代码：

```bash

```

备份插件代码和自有表数据：

```bash
./bin/smart.php xadmin:plugin:backup Todo --with-data -p '<zip-password>'
```

移除插件会自动带数据备份：


备份/恢复只处理 `plugin.json` 声明的 `tables`、`table_prefixes` 或 `plugin.code` 推导出的表前缀；不会读取 release 全局 `backup_tables`。

## 16. 常见问题

| 问题 | 处理方式 |
|------|----------|
| Controller 404 | 检查插件 `composer.json extra.hyperf.config`、Provider 注解扫描、根 Composer require、Composer update 和进程重启 |
| 菜单不显示 | 检查 `plugin.json apps/menus`、`xadmin:menu:sync` 输出、角色菜单授权和菜单缓存 |
| 菜单页面空白 | 检查 `plugin.view_root`、`menus[].view` 和真实 Vue 文件；插件页面不能放到 `web/apps/web-antd` |
| 接口 403 | 检查 Controller `#[Auth]` code、`plugin.json permissions[].code`、节点同步和角色授权 |
| 列表跨租户可见 | 检查表是否有 `tenant_id`，Model `$fillable` 是否包含 `tenant_id`，是否绕过了 CoreMapper 查询 |
| 删除后查不到回收站 | 检查表是否有 `deleted_at`，Model 是否 `use SoftDeletes`，回收站是否调用 `getRecycleList()` |
| 修改前端后生产没变化 | 执行 `composer web:build` 并重新发布；生产包不会运行时编译插件 Vue 页面 |
| ZIP 安装后缺表 | 检查 `plugin.migration_root`、安装环境是否执行初始化/恢复，发布升级是否已生成 DBAL 快照 |

## 17. 进阶：独立前台入口

如果插件不仅有后台管理，还要提供自己的前台登录页和账号体系，需要额外新增：

- 插件自有账号表和账号模型，例如 `TodoAccount`。
- 独立认证控制器，例如 `/todo/account/auth/login/profile/codes/logout/password-crypto`。
- `plugin/Todo/stc/view/auth-entry.ts`，声明 `entry/authBase/loginPath/homePath/routePrefixes/permissionPrefixes/userModel`。
- 前台业务接口显式写 `#[Auth(userModel: TodoAccount::class)]`，不要复用 System 后台 Token。

最小 `auth-entry.ts` 示例：

```ts
export default {
  authBase: '/todo/account/auth',
  entry: 'todo',
  homePath: '/todo/task',
  loginPath: '/todo/login',
  name: 'Todo 应用',
  permissionPrefixes: ['todo.'],
  routePrefixes: ['/todo'],
  userModel: 'Plugin\\Todo\\Model\\TodoAccount',
  userModelIncludes: ['TodoAccount'],
};
```

后台管理插件不需要该文件；只有独立前台入口才需要。

# 智能通道标准化调用

> **会员授权 / 不随开源版源码分发：** 本页为会员授权插件文档，公开文档站展示接口与使用说明，但开源仓不包含该插件源码或 ZIP，安装使用需会员授权或内部交付。

本页面向业务模块开发者，说明如何通过 `plugin/Smart` 统一调用文本对话模型。业务模块只依赖 `SmartConfigService` 和 `SmartGateway`，不要直接读取 `smart_config.api_key`，也不要绕过智能通道自行拼 OpenAI 请求。

## 调用前置条件

| 条件 | 说明 |
|------|------|
| 开放调用 | 管理员已在智能通道接口配置页打开“开放调用”。 |
| 生效配置 | 至少存在一条启用状态、基础地址、接口密钥、模型名称完整的智能通道配置。 |
| 调用方式 | 后端模块通过依赖注入调用 `Plugin\Smart\Service\SmartGateway`，前端不直接代理聊天请求。 |
| 支持标准 | 首期固定支持 OpenAI Responses 接口和 OpenAI Compatible Chat Completions 接口。 |

业务模块展示入口前可以先判断：

```php
use Plugin\Smart\Service\SmartConfigService;

final class DemoService
{
    public function __construct(
        private readonly SmartConfigService $smartConfigs,
    ) {}

    public function smartReady(): bool
    {
        return $this->smartConfigs->isEnabled();
    }
}
```

`isEnabled()` 只有在“开放调用已开启 + 存在生效配置”时才返回 `true`。即使调用方漏判，`SmartGateway` 在真实调用前仍会统一拦截并抛出“智能通道未开放”。

## 推荐入口：chat 文本对话入口

业务模块优先使用 `SmartGateway::chat(array $payload): array`。SmartGateway 对外按通道场景拆分方法，当前只实现 `chat()`；后续图片、语音等通道再增加 `image()`、`audio()` 等同级入口。该方法会自动完成开放状态校验、配置调度、协议适配、调用日志、成功失败计数和敏感字段脱敏。

```php
use Plugin\Smart\Service\SmartGateway;

final class DemoService
{
    public function __construct(
        private readonly SmartGateway $smart,
    ) {}

    public function summarize(string $text): string
    {
        $result = $this->smart->chat([
            'messages' => [
                ['role' => 'system', 'content' => '你是业务摘要助手，只输出 100 字以内中文摘要。'],
                ['role' => 'user', 'content' => $text],
            ],
            'options' => [
                'temperature' => 0.2,
                'max_tokens' => 300,
            ],
        ]);

        return (string)($result['content'] ?? '');
    }
}
```

单轮文本问答也可以使用 `prompt/system` 便捷写法：

```php
$result = $this->smart->chat([
    'system' => '你是 SmartAdmin 助手，回答要简短。',
    'prompt' => '帮我生成一条操作日志摘要',
    'max_tokens' => 200,
]);
```

## 标准传参

### 顶层 payload

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `messages` | array | 与 `prompt` 二选一 | 多轮消息列表，推荐生产代码使用。 |
| `prompt` | string | 与 `messages` 二选一 | 单轮用户输入便捷字段；没有 `messages` 时生效。 |
| `system` | string | 否 | 单轮便捷写法中的系统提示词。 |
| `model` | string | 否 | 临时覆盖配置里的模型名称。通常不传，交给智能通道配置统一管理。 |
| `config_id` | integer | 否 | 指定某条已启用配置；不传时按优先级自动调度。 |
| `options` | object | 否 | 模型参数，见下表。推荐所有模型参数都放在这里。 |
| `include_raw` | boolean | 否 | 是否在返回中包含底层原始响应。仅排障使用，业务逻辑不要依赖。 |

### messages 项

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `role` | string | 否 | 支持 `system`、`developer`、`user`、`assistant`、`tool`；默认 `user`。 |
| `content` | string | 是 | 文本内容。当前智能通道只标准化文本对话，不承诺图片、语音、embedding。 |
| 其它字段 | mixed | 否 | 会作为协议扩展透传，例如 `tool_calls`、`tool_call_id`。 |

### options 常用字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `max_tokens` | integer | 最大输出 token。Responses 标准会自动映射为 `max_output_tokens`。 |
| `max_output_tokens` | integer | Responses 标准最大输出 token。 |
| `temperature` | number | 随机性。Responses 标准如果下游不支持会按适配规则处理。 |
| `top_p` | number | 采样参数。 |
| `response_format` | object | 结构化输出格式，支持 `json_object`、`json_schema` 等 OpenAI 风格写法。 |
| `tools` | array | 工具定义，智能通道只透传，不执行工具。 |
| `tool_choice` | string/object | 工具选择策略。 |
| `metadata` | object | Responses 标准元数据。 |
| `parallel_tool_calls` | boolean | Responses 标准工具并行开关。 |
| `_request_timeout_ms` | integer | 单次调用请求超时覆盖，毫秒。 |
| `_connect_timeout_ms` | integer | 单次调用连接超时覆盖，毫秒。 |

为了简化少量调用，常用模型参数也可以放在顶层 payload；生产代码建议统一放入 `options`，避免业务字段和模型字段混在一起。

## 标准返回

`SmartGateway::chat()` 返回数组：

| 字段 | 类型 | 说明 |
|------|------|------|
| `content` | string | 模型文本输出。 |
| `model` | string/null | 实际响应模型；下游未返回时可能为 `null`。 |
| `finish_reason` | string/null | 停止原因。Compatible 取 `choices[0].finish_reason`，Responses 取 `status`。 |
| `json_object` | object/null | 当 `content` 是 JSON 对象字符串时自动解析；否则为 `null`。 |
| `tool_calls` | array | 模型返回的工具调用列表，智能通道不执行工具。 |
| `usage` | array/object | 下游返回的 token 用量；没有返回时为空数组。 |
| `raw` | object | 只有 `include_raw=true` 时返回。排障字段，业务逻辑不要依赖。 |

示例返回：

```json
{
  "content": "{\"summary\":\"已生成摘要\",\"risk\":\"low\"}",
  "model": "gpt-5.5",
  "finish_reason": "stop",
  "json_object": {
    "summary": "已生成摘要",
    "risk": "low"
  },
  "tool_calls": [],
  "usage": {
    "input_tokens": 128,
    "output_tokens": 24
  }
}
```

## 结构化 JSON 输出

需要稳定 JSON 时，使用 `response_format`，并在提示词中明确“只输出 JSON”：

```php
$result = $this->smart->chat([
    'messages' => [
        ['role' => 'system', 'content' => '只输出 JSON，不要 Markdown。'],
        ['role' => 'user', 'content' => '判断这段文本的风险等级：' . $content],
    ],
    'options' => [
        'temperature' => 0,
        'max_tokens' => 300,
        'response_format' => [
            'type' => 'json_schema',
            'json_schema' => [
                'name' => 'risk_result',
                'schema' => [
                    'type' => 'object',
                    'properties' => [
                        'risk' => ['type' => 'string'],
                        'reason' => ['type' => 'string'],
                    ],
                    'required' => ['risk', 'reason'],
                    'additionalProperties' => false,
                ],
                'strict' => true,
            ],
        ],
    ],
]);

$risk = (string)($result['json_object']['risk'] ?? 'unknown');
```

Responses 标准会把 OpenAI Chat Completions 风格的 `response_format.json_schema` 自动映射到 `text.format`；如果下游对严格 schema 支持不足，适配器会降级为 JSON 对象格式，避免可选格式拖垮主调用。

## 高级入口：DTO 调用

复杂场景可以使用 DTO，但仍应通过 `SmartGateway`：

```php
use Plugin\Smart\Support\ChatRequest;

$request = ChatRequest::fromMessages([
    ['role' => 'system', 'content' => '你是审核助手。'],
    ['role' => 'user', 'content' => $text],
], options: [
    'temperature' => 0,
    'max_tokens' => 400,
]);

$result = $this->smart->chat($request->toArray());
$content = (string)($result['content'] ?? '');
$json = $result['json_object'] ?? null;
```

标准业务入口仍返回数组；需要底层 DTO 时仅在智能通道模块内部适配器场景使用。

## 异常与边界

| 场景 | 行为 |
|------|------|
| 智能通道未开放 | 抛出业务异常“智能通道未开放”。 |
| 没有可用配置 | 抛出业务异常“当前空间未配置可用智能通道接口”。 |
| 指定配置不可用 | 抛出业务异常“指定智能通道配置不存在或未启用”。 |
| 下游 HTTP 失败 | 记录失败日志后抛出下游错误信息。 |
| 下游响应无法解析 | 记录失败日志后抛出“智能通道响应 JSON 解析失败”。 |

调用方建议：

- 后台页面或定时任务可以捕获异常并给出业务提示。
- 不要把 `api_key`、Authorization、Cookie、Token 写入提示词、日志或返回。
- 不要在业务模块保存智能通道配置、模型密钥或 HTTP 客户端状态。
- 当前智能通道固定非流式，返回完整结果后再进入业务流程；如需流式，需要另行设计前端事件和记录边界。

## 测试建议

业务模块接入时至少覆盖：

1. 智能通道未开放时入口隐藏或调用被拦截。
2. 正常调用时能读取 `content`。
3. 结构化输出时能读取 `json_object`。
4. 下游异常时业务流程不会写入半成品数据。

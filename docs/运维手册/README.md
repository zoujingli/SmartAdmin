# 运维手册总览

本章是 [部署运维](../部署运维/README.md) 的补充专题，面向部署与维护人员，覆盖缓存、迁移、日志审计和异常响应等关键运行主题。

如果内容与部署运维存在冲突，以部署运维和当前代码实现为准。

## 运维建议

- 发布前优先执行 `composer release:check`
- 结构变更先执行 `restore --install` dry-run 再正式升级；正式 Phar/SFX 不使用 migrate 作为安装升级入口
- 严格遵循日志脱敏规范，防止敏感信息泄露

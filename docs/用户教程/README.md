# 用户教程

本章面向系统管理员和日常使用者，按真实管理端页面讲解 SmartAdmin 的核心工作流。System 后台页面由 Web 通用运行库和各插件 `plugin.json` 声明的 view 共同提供；插件独立前台入口由插件自己的 `auth-entry.ts` 声明后在前端构建时接入。

## 教程目标

读完本章后，你应该能完成：

- 使用账号登录管理端并理解工作台信息。
- 维护用户、角色、菜单、部门、岗位。
- 为角色分配菜单和按钮权限。
- 理解数据范围如何影响用户能看到的数据。
- 管理文件上传通道、公告和日志。
- 维护租户资料并理解平台侧运维边界。

## 功能入口

| 页面 | 前端路由 | 主要用途 |
|------|----------|----------|
| System 后台登录 | `/auth/login` | 后台用户登录并获取 System Token |
| Project 应用登录 | `/project/login` | ProjectAccount 登录并进入项目应用，入口由 Project 插件声明 |
| 工作台 | `/dashboard/workspace` | 查看待办、系统概览和快捷入口 |
| 数据分析 | `/dashboard/analytics` | 查看日志趋势、系统规模、业务热度 |
| 个人资料 | `/account/profile` | 修改资料、头像、密码和界面偏好 |
| 用户管理 | `/system/user` | 用户 CRUD、状态、密码重置、角色/部门/岗位分配 |
| 角色管理 | `/system/role` | 角色 CRUD、权限树授权、角色用户查看 |
| 菜单管理 | `/system/menu` | 菜单树、按钮权限、路由元数据 |
| 部门管理 | `/system/dept` | 组织树、部门成员、排序和状态 |
| 岗位管理 | `/system/post` | 岗位 CRUD、状态和前端导出 |
| 租户管理 | `/system/tenant` | 平台侧维护租户资料和状态 |
| 通知中心 | `/system/notice` | 公告发布、收件箱、已读和归档 |
| 文件管理 | `/system/file` | 文件记录、上传通道、下载、去重 |
| 操作日志 | `/system/logs/action` | 接口调用、请求响应、分析和清理 |
| 变更日志 | `/system/logs/change` | 业务对象变更明细和关联操作回看 |
| 系统参数 | `/system/setting` | 维护运行参数和界面相关配置 |
| 系统数据 | `/system/data` | 系统配置、能力矩阵、缓存清理 |

## 核心工作流

```mermaid
flowchart LR
  Login["登录"] --> Dept["维护部门/岗位"]
  Dept --> Menu["维护菜单/按钮"]
  Menu --> Role["创建角色并授权"]
  Role --> User["创建用户并分配关系"]
  User --> Verify["用普通用户验证权限"]
  Verify --> Ops["日志/公告/文件/租户运维"]
```

## 使用前提

- 已完成 [本地启动](../快速开始/本地启动.md) 或已有可访问的部署环境。
- 当前账号具备相应菜单权限。
- 插件清单、菜单和权限节点已经同步。
- 如果是多租户环境，当前账号已建立正确租户上下文。

## 页面共性

多数管理页面都包含：

- 搜索区：按名称、编码、状态、时间等筛选。
- 表格区：分页、状态展示、操作列。
- 工具栏：新增、批量删除、导出、刷新。
- 回收站：恢复或彻底删除软删除数据。
- 统计区：展示模块数量、状态和趋势。
- 操作日志：记录关键写操作。

如果某个页面行为异常，先检查权限码、数据范围、接口响应和浏览器 Network。

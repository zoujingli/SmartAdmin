# 文件、通知与日志

这三个模块解决后台系统常见的“资源存储、站内通知、审计追踪”问题。文件模块负责上传记录和通道配置，公告模块负责从管理端发布到用户收件箱，日志模块负责记录请求、操作和字段变更。

## 文件管理

页面：`/system/file`

文件管理维护上传后的文件记录，支持列表、回收站、详情、下载、编辑、删除、恢复、统计、上传通道配置和文件去重。

上传能力由 `/system/file/upload/*` 系列接口提供，支持上传场景、运行时配置、预处理、中转上传、分片上传、远端分片签名、完成和终止。

### 文件记录字段

| 字段 | 说明 |
|------|------|
| `scene` | 上传场景，用于区分头像、附件、业务文件等 |
| `driver` | 存储驱动，例如 local、oss、cos、qiniu、alist |
| `url` | 可访问地址 |
| `hash` | 文件哈希，用于去重 |
| `suffix` | 文件后缀 |
| `storage_mode` | 存储模式 |
| `origin_name` | 原始文件名 |
| `object_name` | 存储对象名 |
| `storage_path` | 存储路径 |
| `mime_type` | MIME 类型 |
| `size_byte` / `size_info` | 文件大小，分别用于计算和展示 |

### 上传通道配置

页面内的“上传通道配置”按驱动分组维护。配置保存接口会对敏感字段脱敏记录日志，文档和日志中都不应出现明文密钥。

| 驱动 | 典型配置 | 使用场景 |
|------|----------|----------|
| local | 本地路径、访问域名、文件大小限制 | 开发环境、内网部署、小规模私有化 |
| alist | 服务地址、Token、根目录 | 已有 Alist 统一文件网关 |
| qiniu | AccessKey、SecretKey、Bucket、Domain、Region | 七牛云对象存储 |
| oss | AccessKeyId、AccessKeySecret、Bucket、Endpoint、Domain | 阿里云 OSS |
| cos | SecretId、SecretKey、Bucket、Region、Domain | 腾讯云 COS |

### 推荐上传流程

1. 前端调用 `GET /system/file/upload/runtime` 获取当前上传场景与运行时限制。
2. 调用 `POST /system/file/upload/prepare` 创建上传会话，服务端决定驱动、路径、分片策略。
3. 小文件使用 `POST /system/file/upload/relay` 中转上传。
4. 大文件按分片上传，必要时先调用 `part-sign` 获取远端分片签名。
5. 全部分片完成后调用 `complete` 落库并返回文件记录；失败时调用 `abort` 终止会话。

### 操作建议

- 开发环境优先使用 local 驱动，确认上传链路稳定后再切换对象存储。
- 生产环境必须把上传目录或对象存储纳入备份策略。
- 文件去重前先确认业务是否允许多条记录引用同一物理文件。
- 彻底删除文件记录前，应确认没有业务表仍引用该文件 URL 或文件 ID。

## 通知中心

页面：`/system/notice`

通知中心分为公告管理端和我的收件箱：

- 管理端维护公告草稿、发布、状态、收件人、回收站。
- 收件箱支持未读数、标记已读、全部已读、归档和全部归档。
- 公告发布后会生成用户维度的接收记录，读取状态不影响公告主记录。

### 公告字段

| 字段 | 说明 |
|------|------|
| `title` | 公告标题，列表和收件箱主展示字段 |
| `content` | 公告正文 |
| `level` | 公告级别，用于区分普通、重要等展示样式 |
| `status` | 草稿、启用、禁用等状态 |
| `published_at` | 发布时间 |
| `expired_at` | 过期时间，过期公告不应继续作为有效通知展示 |
| `link` | 可选跳转链接 |

### 发布流程

1. 管理员创建公告草稿，填写标题、正文、级别、有效期和接收范围。
2. 编辑确认后执行发布操作，服务端生成用户维度接收记录。
3. 用户进入收件箱，系统按当前用户查询可见公告和未读数。
4. 用户标记已读或归档，只影响自己的接收记录。
5. 管理员禁用或删除公告，会影响公告主记录的展示和管理状态。

### 常见边界

- 公告管理权限和收件箱权限不同：管理端需要菜单权限，收件箱只要求登录。
- 公告发布后的接收关系应以服务端记录为准，不建议前端临时计算。
- 已归档不是删除，只是当前用户不再在默认收件箱中查看。

## 日志管理

页面：`/system/logs/action`、`/system/logs/change`

日志管理用于定位操作、异常和审计问题。操作日志支持列表、详情、删除、清空、统计、前端导出、分析报告和近实时指标；变更日志支持独立列表、详情和关联操作回看。

日志来源包括：

- `LogsMiddleware` 记录全局请求信息。
- `#[Logger]` 注解记录关键写操作。
- 模型审计规则和 Service 手动记录业务变更。

### 日志字段

| 字段 | 说明 |
|------|------|
| `username` | 操作人 |
| `method` | HTTP 方法 |
| `router` | 请求路由 |
| `name` | 操作名称，通常来自 `#[Logger]` |
| `remark` | 操作备注 |
| `ip` / `ip_location` | 请求 IP 和归属地 |
| `os` / `browser` | 客户端环境 |
| `request_data` | 请求数据，按规则脱敏和截断 |
| `response_code` | 响应业务码 |
| `response_data` | 响应数据，按规则脱敏和截断 |
| `action_id` | 变更日志关联的操作日志 ID |
| `change_values` | 变更日志字段明细 |
| `change_remark` | 变更日志可读摘要 |

### 分析与清理

- `statistics` 用于页面统计卡片，适合展示总量、成功/失败、近期趋势。
- `metrics` 用于近实时指标，适合工作台或监控面板轮询。
- `analysis` 用于聚合分析报告，适合排查高频失败接口和异常趋势。
- `clean` 清理过期日志，适合定时任务。
- `archive` 归档日志，适合把审计数据转移到低成本存储。
- `clear` 是高风险操作，生产环境建议限制权限并保留外部备份。

## 常见问题

- 上传失败：先检查上传通道配置，再检查文件大小、后缀、MIME 和分片大小。
- 公告未收到：确认公告已发布且接收人范围包含当前用户。
- 日志没有敏感字段：这是预期行为，密码、Token、Secret、Key 等字段会脱敏。
- 日志列表太大：启用归档和过期清理策略，前端导出大范围数据时按分页串行拉取，避免影响在线库。
- 上传配置保存后不生效：检查是否存在运行时缓存，必要时清理业务缓存并重新获取 runtime 配置。

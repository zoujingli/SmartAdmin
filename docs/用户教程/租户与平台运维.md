# 租户与平台运维

## 租户管理

页面：`/system/tenant`

租户管理属于平台运维能力，用于维护租户资料、联系人、状态、有效期和备注。对应后端 `plugin/System` 模块。

支持能力：

- 租户列表、回收站、详情。
- 新增、编辑、删除、彻底删除、恢复。
- 启用/禁用租户。
- 租户统计和前端导出。

## 租户隔离

普通业务表如果包含 `tenant_id` 且模型 `$fillable` 包含该字段，会自动启用租户范围。登录后用户服务会建立租户上下文，普通查询依赖 `TenantContext` 和 `CoreModel` 的全局作用域。

## 平台运维入口

平台运维菜单包括：

- 租户管理：维护租户生命周期。
- 通知中心：面向平台或用户发布公告。
- 日志管理：查看请求和操作审计。
- 文件管理：维护上传通道和文件记录。
- 系统参数：维护运行参数和界面相关配置。
- 系统数据：查看系统能力、配置和缓存。

## 使用建议

- 平台管理员负责租户生命周期和全局配置。
- 租户内管理员负责本租户用户、角色、菜单和组织数据。
- 不要把普通业务查询写成绕过租户上下文的原生查询；确需原生查询时必须显式处理租户和数据范围。

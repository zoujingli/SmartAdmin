# 登录与工作台

## 登录流程

打开 `http://localhost:5666/auth/login`，输入默认账号 `admin/admin`。登录成功后前端调用 `/system/auth/login` 获取访问令牌、用户信息、`auth_user_model` 和权限码。

在线演示环境为 `https://smart.thinkadmin.top`，默认账号和密码同样是 `admin/admin`。该环境使用 `APP_ENV=demo`，允许浏览、查询和 Project 普通业务流程体验；系统基础资料、账号权限、全局配置、微信配置、删除、恢复和禁用类操作会被拦截并提示“演示环境禁止修改关键数据”。

登录后的前端请求会自动携带 Token；当接口返回 Token 过期时，前端请求层会调用 `/system/auth/refresh` 刷新令牌。用户主动退出时调用 `/system/auth/logout`。

Project 应用使用独立入口 `http://localhost:5666/project/login`，登录主体为 `ProjectAccount`，接口范围为 `/project/**`。System 后台账号不能直接进入 Project 应用接口；如需在后台管理 Project 数据，请从后台菜单 `/system/project` 进入。

Project 登录页和入口配置由 Project 插件自己的 `stc/view/auth-entry.ts` 声明，Web 前端会在编译期扫描读取。登录后页面可读取共享的界面元信息，但任务、产品、测试和报表等业务数据仍只能使用 Project Token 访问 `/project/**`。

## 个人资料

个人资料页面位于 `/account/profile`，对应后端 `/system/auth/profile`、`/system/auth/password` 和 `/system/auth/preferences`：

- 修改昵称、头像、邮箱、手机号等资料。
- 修改当前用户密码，旧密码和新密码不会写入操作日志明文。
- 保存界面偏好，后端按白名单过滤后写入用户扩展配置。

## 工作台

工作台页面位于 `/dashboard/workspace`，数据来自系统数据接口：

- `/system/data/index` 提供用户、角色、菜单、部门、岗位、节点、日志等统计。
- `/system/data/todos` 提供待处理事项，例如异常日志、待清理文件、即将到期租户。
- `/system/data/capabilities` 展示当前系统能力矩阵。

## 数据分析

数据分析页面位于 `/dashboard/analytics`，主要聚合日志和系统规模数据。它不是单独的业务表，而是基于 `System\DataService`、`LogsActionService` 和 `LogsChangeService` 的统计能力展示系统运行状态。

## 常见问题

- 登录提示账号密码错误：这是业务失败，后端按项目规范返回 `500`，不是 `401`。
- 登录后菜单为空：检查角色是否分配菜单和权限节点，必要时执行节点同步命令。
- Project 登录后提示元信息或权限码未登录：先强制刷新旧页面并重新登录；仍失败时检查当前地址是否为 `/project/login`，以及后端 Project 账号是否启用。
- 修改偏好不生效：清理浏览器缓存并重新登录，确认 `/system/auth/preferences` 返回成功。

# 缓存、日志与上传

缓存、日志和上传是生产环境最容易被忽略的三类运行态数据。它们不只是配置项，还会影响权限生效、登录体验、审计合规、磁盘容量和文件可用性。

## 缓存

本地开发可以使用 File 缓存，生产和多实例环境推荐 Redis。权限缓存、用户上下文缓存、上传会话和列表快照都可能依赖缓存一致性。

清理业务缓存：

```text
POST /system/data/clear-cache
```

## 缓存策略

| 场景 | 推荐方案 | 注意事项 |
|------|----------|----------|
| 本地开发 | File 或单机 Redis | 便于调试，可随时清理 |
| 单机生产 | Redis | 避免 File 缓存在 Swoole 多 worker 下不一致 |
| 多实例生产 | 独立 Redis 集群或托管 Redis | 所有实例必须共享缓存 |
| 权限变更 | 清理权限缓存或重新登录 | 菜单和按钮权限可能被前端或后端缓存 |
| 上传会话 | Redis | 分片上传跨请求依赖会话一致性 |

### 缓存排查

- 菜单或按钮授权后不生效：重新登录，必要时清理业务缓存。
- 修改 `.env` 后不生效：重启 Swoole，不要只刷新前端。
- 多实例表现不一致：确认所有实例连接同一个 Redis 和同一套配置。
- 上传分片丢失：检查上传会话缓存 TTL、Redis 连接和 worker 重启情况。

## 日志

日志包括请求日志、操作日志、异常日志和模型变更日志。生产环境应关注：

- 日志目录磁盘容量。
- 敏感字段脱敏。
- 过期日志清理策略。
- 清理、归档和恢复等操作权限。

相关接口见 [日志公告接口](../接口参考/日志公告接口.md)。

### 日志分类

| 类型 | 来源 | 用途 |
|------|------|------|
| 请求日志 | 全局请求日志中间件 | 接口访问、状态码、耗时、客户端环境 |
| 操作日志 | `#[Logger]` 注解 | 写操作审计、清空、发布等关键动作 |
| 模型变更日志 | Model `$logRules` | 字段级变更追踪 |
| 异常日志 | Hyperf 日志系统 | 程序错误、外部服务异常、未捕获异常 |

### 脱敏规则

以下字段不得明文入库或写入日志：

- 密码、旧密码、新密码。
- Token、Authorization、Cookie。
- Secret、Key、AccessKey、SecretKey。
- 上传签名、临时凭证。
- 其他业务上的个人敏感信息。

脱敏规则应支持点路径，例如 `drivers.oss.access_secret`。记录大 JSON body 时只保留限制长度内的预览，超出部分使用 `...` 截断，并在写入前脱敏。

### 日志保留

| 操作 | 建议 |
|------|------|
| 在线日志 | 保留最近 30 到 90 天，按业务合规要求调整 |
| 归档日志 | 转移到低成本存储，保留索引和备份 ID |
| 清空日志 | 仅限平台管理员，操作前确认备份 |

### 磁盘监控

生产环境至少监控：

- `runtime/logger` 增长速度。
- 上传目录容量。
- `runtime/backup` 运行备份目录容量。
- 归档目录容量。
- PHP/Swoole 错误日志。

## 上传

上传配置由文件管理页面维护，支持本地、OSS、COS、Qiniu、Alist。生产环境建议：

- 限制文件大小、后缀和 MIME。
- 远端存储密钥只放在后端配置和数据库密文字段，不写入前端。
- 开启分片上传时合理配置分片大小。
- 定期检查重复文件和孤儿文件。

## 上传驱动选择

| 驱动 | 优点 | 风险 |
|------|------|------|
| local | 配置简单、适合开发和内网 | 多实例共享困难，磁盘扩容和备份压力大 |
| OSS/COS/Qiniu | 可扩展、适合公网和大文件 | 依赖云厂商配置、域名、签名和权限 |
| Alist | 可接入已有文件网关 | 需要维护 Alist 可用性和权限 |

## 上传安全

- 服务端必须校验文件大小、后缀、MIME 和场景。
- 前端传入的文件名不能直接作为存储对象名。
- 对象存储密钥只保存在服务端配置中，前端只使用临时签名或公开 URL。
- 下载接口应校验文件管理权限，业务下载接口还应校验业务数据权限。
- 图片、HTML、SVG 等可执行或可嵌入内容需要额外关注 XSS 和 MIME 嗅探。

## 分片上传运维

分片上传涉及会话、临时分片、最终合并和清理：

1. `prepare` 创建上传会话并写入缓存。
2. 前端按分片大小上传或获取远端签名。
3. 服务端记录已上传分片和校验信息。
4. `complete` 完成合并、落库和清理临时状态。
5. `abort` 终止会话并清理临时文件或远端分片。

运维关注点：

- 上传会话 TTL 不宜过短，避免大文件上传中途失效。
- 临时分片目录要定期清理。
- 对象存储分片上传失败后要 abort，避免产生悬挂分片。
- 分片大小要结合前端网络、PHP 上传限制、Nginx 限制和对象存储限制一起配置。

## 备份清单

| 数据 | 是否需要备份 | 说明 |
|------|--------------|------|
| 数据库 | 是 | 业务数据、用户、权限、文件记录、公告、日志 |
| local 上传目录 | 是 | 本地驱动下文件实体在服务器磁盘 |
| 对象存储 Bucket | 是 | 依赖云厂商版本控制、生命周期或跨区域备份 |
| `.env` | 是 | 生产配置和密钥，需加密保存 |
| `runtime/keys` | 是 | 密码传输 RSA 私钥和内部 OpenSSL 配置，由系统自动维护；多实例部署需共享或随应用一致分发 |
| `runtime/backup` | 是 | release 运行备份，默认保存结构 + 必要数据，`--with-data` 保存结构 + 全量数据 |
| `runtime/logger` | 视合规要求 | 审计要求高的场景需要归档 |

## 故障处理

| 现象 | 处理 |
|------|------|
| 权限修改不生效 | 重新登录、清理业务缓存、检查节点同步 |
| 日志表过大 | 先归档，再按保留期清理，避免直接清空 |
| 上传 413 | 检查 Nginx、PHP、前端和系统上传大小限制 |
| 对象存储 403 | 检查密钥、Bucket 权限、Region、Endpoint 和临时签名 |
| 文件 URL 访问 404 | 检查 Domain、object_name、storage_path、CDN 回源 |
| 分片完成失败 | 检查 upload_id、分片数量、分片哈希、缓存 TTL 和临时目录 |
| 密码加密参数获取失败 | 检查 OpenSSL 扩展、`runtime/keys` 写权限；系统会自动生成 RSA 私钥和内部 `openssl.cnf`，不需要手工配置证书 |

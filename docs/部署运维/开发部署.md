# 开发部署

开发部署用于本地调试后端、前端和 docs。它不追求高可用，但必须尽量接近生产运行方式：后端通过 Swoole 启动，前端通过 Vite 开发服务，docs 通过独立静态站预览。

## 前置条件

请先确认 [环境准备](../快速开始/环境准备.md) 已完成：

- PHP 和 Swoole 运行时可用。
- MySQL 数据库已创建。
- Redis 可用或本地配置可使用 File 缓存。
- Node.js 和 pnpm 可用。
- `.env` 已从 `.env.example` 复制并填写。

## 后端

```bash
composer install
cp .env.example .env
composer setup
sh bin/start-swoole start
```

开发监听可以使用：

```bash
composer watch
```

### 初始化说明

`composer setup` 通常会完成：

- 数据库迁移或初始化。
- 菜单同步。
- 权限节点同步。
- 默认管理员和基础数据准备。

如果你只改菜单或权限注解，可以单独执行：

```bash
sh bin/start-swoole xadmin:menu:sync --details
sh bin/start-swoole xadmin:node:sync --details
```

### 后端调试

| 场景 | 建议 |
|------|------|
| 改 Controller/Service | 重启 Swoole 进程 |
| 改配置文件 | 重启 Swoole |
| 改注解 | 重启并同步节点 |
| 改迁移 | 重新执行迁移或 fresh 初始化 |
| 改公共 Library | 运行相关单测 |

## 前端

```bash
cd web
pnpm install
pnpm dev:antd
```

前端 API 地址由环境变量和运行时配置决定。开发阶段如果跨域异常，可以切换到 Vite 代理模式。

### 前端调试

| 场景 | 检查 |
|------|------|
| 登录接口 404 | API 地址、Vite 代理、后端端口 |
| 登录后空白 | 菜单 route、component、前端路由 |
| 按钮不显示 | 权限码、角色授权、`/system/auth/codes` |
| 表单提交失败 | 字段名、必填字段、后端校验 |
| 构建失败 | TypeScript 类型、依赖版本、锁文件 |

## 文档

```bash
composer docs:serve
```

文档服务只托管 `docs/` 目录，默认端口 `18100`。检查 docsify 配置：

```bash
composer docs:check
```

## 开发验证

按改动范围选择：

| 改动 | 命令 |
|------|------|
| 文档 | `composer docs:check` |
| PHP 文件语法 | `./bin/swoole-macos-a64 -l <file>` |
| 后端公共逻辑 | `composer analyse`、`composer test` |
| 前端页面/API | `composer web:build` |
| 发布升级 | `composer release:check` |

## 本地数据重置

开发过程中如果需要重置数据，请先确认不是生产库或共享库，再执行初始化相关命令。重置会影响用户、角色、菜单、权限节点、文件记录和业务数据，必要时先备份。

## 常见问题

| 问题 | 处理 |
|------|------|
| Swoole 启动失败 | 看终端错误和 `runtime/logger` |
| 数据库表缺失 | 重新执行 `composer setup` 或迁移 |
| 菜单缺失 | 同步菜单和节点后重新登录 |
| docs 页面旧内容 | 强刷浏览器，必要时清理缓存 |
| 前端依赖异常 | 删除本地缓存后重新 `pnpm install`，不要手改 lock 文件 |

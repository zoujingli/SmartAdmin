# 环境准备

SmartAdmin 后端运行在 Hyperf 3.2 + Swoole 6，前端运行在 Vue 3 + Vite + pnpm Monorepo。开发环境建议使用 macOS 或 Linux，Windows 推荐使用 WSL2。

## 推荐环境

| 场景 | 推荐 |
|------|------|
| 本地开发 | macOS 或 Linux，使用项目内置 Swoole 运行时 |
| Windows 开发 | WSL2 + Linux 发行版 |
| 生产部署 | Linux + PHP/Swoole 或项目构建产物 |
| 文档预览 | 任意可运行 PHP 内置服务器的环境 |

## 基础依赖

| 依赖 | 建议版本 | 说明 |
|------|----------|------|
| PHP | `>= 8.4` | 需要 `swoole`、`pdo_mysql`、`redis`、`openssl`、`zlib`、`bcmath` |
| MySQL | `>= 8.0` | 主数据库 |
| Redis | `>= 6.0` | 生产缓存、Token 状态、上传会话推荐 |
| Node.js | `>= 20.19` | 前端工程要求 |
| pnpm | `>= 10` | 前端包管理 |

## PHP 扩展

至少确认：

| 扩展 | 用途 |
|------|------|
| `swoole` | Hyperf HTTP Server |
| `pdo_mysql` | MySQL 数据库访问 |
| `redis` | Redis 缓存、Token 状态、上传会话 |
| `openssl` | 加密、签名、HTTPS 相关能力 |
| `zlib` | 压缩和快照相关能力 |
| `bcmath` | 高精度数值处理 |
| `json` | JSON 编解码 |
| `mbstring` | 多字节字符串 |

使用项目内置运行时时，可以通过下列命令查看版本和扩展：

```bash
./bin/swoole-macos-a64 -v
./bin/swoole-macos-a64 -m
```

## 后端运行时

仓库提供 `bin/smart`，Composer 脚本会优先使用项目内置 Swoole 运行时。如果当前平台没有匹配运行时，会回退到本机带 Swoole 扩展的 `php`，此时需要本机 PHP 安装对应扩展。

```bash
./bin/swoole-macos-a64 -v
./bin/swoole-macos-a64 -m
```

如果本机平台没有对应的内置运行时，建议优先安装符合版本要求的 PHP 和 Swoole，而不是修改项目启动脚本。

## 数据库和缓存

1. 创建 MySQL 数据库，字符集建议 `utf8mb4`。
2. 确认 `.env` 中数据库账号有建表、改表、索引权限。
3. 本地开发可以先使用 File 缓存。
4. 多实例或生产环境推荐 Redis，避免 Token、权限缓存、上传会话在实例间不一致。

### MySQL 建议

| 项目 | 建议 |
|------|------|
| 字符集 | `utf8mb4` |
| 排序规则 | 按项目和 MySQL 版本统一 |
| 权限 | 本地开发账号需要建表、改表、索引、读写权限 |
| 时区 | 与应用和数据库保持一致 |
| 备份 | 生产环境必须有可恢复备份 |

### Redis 建议

Redis 在生产环境建议独立部署，并设置合理密码、网络访问限制和持久化策略。本地开发可以临时使用 File 缓存，但多 worker、多实例、上传分片会话更推荐 Redis。

## 初始化

如果还没有项目目录，推荐用 Composer 创建正式发布版项目：

```bash
composer create-project zoujingli/smartadmin SmartAdmin
cd SmartAdmin
```

如果是 GitHub 克隆源码或创建项目时使用了 `--no-install`，再执行依赖安装：

```bash
composer install
```

然后复制环境文件并初始化数据库、菜单和权限节点：

```bash
cp .env.example .env
composer setup
```

`composer setup` 会执行迁移，并同步菜单和权限节点。菜单统一来源于各插件 `plugin.json`；权限节点还会汇总 `#[Auth]` 注解。新增或调整插件清单后，在源码/CI 阶段执行菜单、节点 dry-run 确认同步结果。

初始化前要确认 `.env` 指向的是本地开发库。不要在生产库或共享测试库上随意执行重置类初始化。

## 前端准备

```bash
cd web
pnpm install
```

如果 pnpm 安装依赖慢，优先配置本地镜像源，不要手工修改 `pnpm-lock.yaml`。

### Node 和 pnpm 检查

```bash
node -v
pnpm -v
```

前端依赖问题优先通过 pnpm 配置和缓存处理，不建议手工编辑 lock 文件或混用 npm/yarn 安装同一工作区。

## 文档服务准备

文档是独立 docsify 静态站，不需要 Node 构建。只要本机能执行 PHP，就可以运行：

```bash
composer docs:serve
```

默认访问 `http://127.0.0.1:18100`。

## 端口规划

| 服务 | 默认端口 | 说明 |
|------|----------|------|
| 后端 API | `9501` | Hyperf/Swoole |
| 前端 dev server | `5666` | Vite 管理端 |
| docs 预览 | `18100` | PHP 内置静态服务 |

端口冲突时先确认占用进程，再调整对应配置。docs 端口来自 `composer docs:serve` 脚本。

## 环境检查命令

```bash
composer --version
php -v
php -m
mysql --version
redis-server --version
node -v
pnpm -v
```

如果使用项目内置 Swoole 运行时，以 `./bin/swoole-macos-a64 -v` 和 `./bin/swoole-macos-a64 -m` 为准。

## 常见问题

| 问题 | 处理 |
|------|------|
| `composer create-project` 或 `composer install` 失败 | 检查 PHP 版本、扩展和 Composer 镜像 |
| Swoole 扩展缺失 | 使用项目内置运行时或安装匹配版本扩展 |
| MySQL 连接失败 | 检查 `.env`、账号密码、库名和网络 |
| Redis 不可用 | 本地可临时切 File 缓存，生产应修复 Redis |
| pnpm 安装慢 | 配置镜像源，不要手工改 lock |
| docs 无法启动 | 确认 PHP 可执行，端口 `18100` 未占用 |

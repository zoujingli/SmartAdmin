# 本地启动

本页按最小路径启动后端、前端和文档站。启动前请先完成 [环境准备](./环境准备.md)。

## 启动顺序

推荐按以下顺序启动，便于定位问题：

1. 安装 PHP 依赖和前端依赖。
2. 配置 `.env`。
3. 初始化数据库、菜单和权限节点。
4. 启动后端 Swoole。
5. 启动前端开发服务。
6. 启动 docsify 文档站。

## 创建项目与安装依赖

如果还没有项目目录，推荐直接使用已发布的 Composer 包创建正式版项目：

```bash
composer create-project zoujingli/smartadmin SmartAdmin
cd SmartAdmin
```

`composer create-project` 会同时安装后端依赖。如果你是通过 GitHub 克隆源码或使用了 `--no-install`，再手动执行：

```bash
composer install
```

然后安装前端依赖：

```bash
cd web
pnpm install
cd ..
```

如果 `composer create-project` 或 `composer install` 因 PHP 扩展失败，先回到 [环境准备](./环境准备.md) 对照检查 Swoole、PDO、Redis、mbstring、openssl 等扩展。

## 配置环境

复制环境文件后修改数据库、Redis、服务端口和前端地址：

```bash
cp .env.example .env
```

重点确认：

| 配置 | 说明 |
|------|------|
| 数据库连接 | MySQL 主机、端口、库名、账号、密码 |
| Redis 连接 | 缓存、会话、上传会话等能力可能依赖 Redis |
| Swoole 端口 | 默认后端服务端口为 `9501` |
| 登录令牌密钥 | 本地可使用默认兼容配置；生产应按部署安全要求替换为独立密钥配置 |
| 上传配置 | 本地开发建议先使用 local 驱动 |

## 初始化数据

首次启动前执行项目初始化脚本：

```bash
composer setup
```

该命令会按项目脚本完成数据库初始化、菜单同步和权限节点同步。若你只想单独同步菜单或节点，可以执行：

```bash
sh bin/start-swoole xadmin:menu:sync --details
sh bin/start-swoole xadmin:node:sync --details
```

初始化后建议确认以下数据存在：

- `system_user` 中有默认管理员。
- `system_menu` 中有系统管理、用户、角色、菜单、部门、岗位、文件、日志、公告等菜单。
- `system_node` 中有 Controller 注解和菜单 code 生成的权限节点。

## 后端启动

```bash
sh bin/start-swoole start
```

启动后访问后端 API：

```text
http://127.0.0.1:9501
```

如果端口被占用，修改 `.env` 中的 `APP_WORKER_PORT`，或关闭占用 `9501` 的进程。

后端进程以前台方式运行；本地停止使用 `Ctrl+C`，配置、Provider、注解或插件 view 路径变更后重新执行启动命令。

后端启动失败时先看终端错误，再看 `runtime/logger`。如果修改了 `.env`、Provider、注解、`plugin.json` 或插件 view 路径，建议重启后端和前端 dev server，而不是只刷新浏览器。

## 前端启动

```bash
cd web
pnpm install
pnpm dev:antd
```

访问管理端：

```text
http://localhost:5666
```

前端默认读取 `web/apps/web-antd` 的环境配置。开发时可以选择直连后端或使用 Vite 代理；代理模式会统一请求 `/api/<真实接口前缀>`，再由 Vite 转发到后端真实路径，避免把插件私有前缀硬编码进 Web 壳。

### 前端联调检查

| 现象 | 检查点 |
|------|--------|
| 登录请求 404 | API 地址是否指向 `http://127.0.0.1:9501` 或代理路径是否正确 |
| 登录后空白 | 前端路由和后端菜单 component 是否匹配 |
| 菜单缺失 | 当前用户角色是否授权，`plugin.json`、菜单和节点是否同步 |
| 按钮缺失 | 前端按钮权限码是否与后端 `#[Auth]` code 一致 |
| 请求 401 | Token 是否过期，刷新接口是否可用 |
| 请求 403 | 当前用户是否具备接口权限 |

## 默认账号

- 用户名：`admin`
- 密码：`admin`

忘记本地管理员密码时，可以使用控制台命令重置。命令不会输出明文密码，数据库仍保存 `password_hash()` 哈希值。

```bash
# 交互式输入新密码
sh bin/start-swoole xadmin:pass:reset

# 重置为初始化默认密码 admin
sh bin/start-swoole xadmin:pass:reset --default
```

## 文档站启动

```bash
composer docs:serve
```

访问：

```text
http://127.0.0.1:18100
```

文档只读取 `docs/` 目录，不依赖后端服务；如果需要检查 docsify 导航规则，执行：

```bash
composer docs:check
```

docs 服务使用 PHP 内置静态服务器，以 `docs/` 为站点根目录。它只适合本地预览，生产环境请使用 Nginx、对象存储静态站或 Pages 平台托管。

## 开发验证

本地改动完成后，按改动范围选择验证命令：

| 改动范围 | 建议命令 |
|----------|----------|
| 只改文档 | `composer docs:check` |
| 改 PHP 单文件 | `./bin/swoole-macos-a64 -l <file>` |
| 改后端公共逻辑 | `composer analyse`、`composer test` |
| 改前端页面或 API | `composer web:build` |
| 改发布构建 | `composer release:check` |

## 常见问题排查

- 端口占用：检查 `9501`、`5666`、`18100`。
- 数据库连接失败：检查 `.env` 与 MySQL 用户权限。
- 菜单为空：重新执行 `composer setup`，或用 `xadmin:menu:sync --dry-run --json` 检查插件 `plugin.json` 菜单声明。
- 登录后没有按钮：执行 `sh bin/start-swoole xadmin:node:sync --details` 同步权限节点。
- 前端接口跨域：确认后端 CORS 中间件与前端 API 地址配置。
- 上传失败：先使用 local 驱动验证链路，再检查对象存储密钥、Bucket、Region、Domain。
- 代码修改无效：确认后端是否已重启，前端是否热更新成功。
- docs 点击无反应：执行 `composer docs:check` 检查 `_sidebar.md`、顶部模块导航脚本和内部链接规则。

# 安全与漏洞披露

安全问题优先级高于普通功能和文档问题。请不要在公开 issue、公开 PR 或社交平台直接发布可利用细节，尤其是涉及认证绕过、权限绕过、任意文件访问、密钥泄露、SQL 注入、远程代码执行等问题。

## 安全原则

- 优先保护用户数据与访问凭证。
- 默认最小权限与最小暴露面。
- 安全问题优先修复，必要时先缓发细节。
- 敏感字段默认脱敏，不进入日志、响应、导出和文档。
- 权限校验以后端 `#[Auth]` 为准，前端隐藏按钮不是安全边界。

## 漏洞披露建议流程

1. 先私下联系维护者反馈漏洞细节。
2. 提供复现步骤、影响范围和修复建议。
3. 维护者确认影响版本和修复方案。
4. 发布修复版本或补丁。
5. 等待用户有合理升级窗口后，再公开技术细节。

## 报告内容建议

请尽量包含：

| 内容 | 说明 |
|------|------|
| 漏洞类型 | 例如权限绕过、SQL 注入、XSS、任意文件访问 |
| 危害等级 | 你判断的影响程度和理由 |
| 影响版本 | commit、tag、分支或部署方式 |
| 复现条件 | 是否需要登录、需要什么角色、是否需要租户环境 |
| 复现步骤 | 尽量最小化、可重复 |
| PoC | 如有请脱敏，避免包含真实密钥和用户数据 |
| 影响范围 | 可访问的数据、可执行的操作、是否跨租户 |
| 修复建议 | 可选，但有助于快速定位 |

## 重点关注区域

| 区域 | 风险 |
|------|------|
| 认证和 Token | Token 泄露、刷新绕过、登出失效 |
| 权限和菜单 | `#[Auth]` 缺失、权限码复用、前端权限误当安全边界 |
| 数据范围 | 列表以外的详情、更新、删除、导出越权 |
| 租户隔离 | `tenant_id` 缺失、上下文绕过、超级管理员误用 |
| 日志 | 明文密码、Token、Secret、Cookie、上传签名入库 |
| 上传 | 任意文件上传、MIME 绕过、对象存储签名泄露 |
| 发布升级 | 受管表误配置、破坏性 SQL 未确认、备份不可恢复 |
| 前端配置 | `_app.config.js` 暴露敏感配置 |

## 临时缓解建议

在修复发布前，维护者可能建议用户采取临时措施：

- 禁用受影响接口或菜单。
- 收紧角色授权。
- 清理或轮换泄露的 Token、密钥和对象存储凭证。
- 暂停高风险上传场景。
- 禁止执行发布 `--force`。
- 增加 Nginx 或 WAF 限制。
- 回滚到不受影响版本。

临时缓解不等于最终修复，最终仍应发布补丁和升级说明。

## 修复要求

安全修复 PR 应尽量包含：

- 最小修复范围。
- 回归测试或复现用例。
- 文档或发布说明。
- 是否需要用户轮换密钥、清理缓存、重新登录或执行升级命令。
- 是否影响接口响应、权限码、菜单和前端页面。

## 注意事项

- 请勿在公开 issue 直接发布可利用细节。
- 涉及密钥、令牌、账号信息时请先做脱敏处理。
- 不要上传生产数据库、日志、`.env`、对象存储配置或用户文件。
- 不要在 PoC 中破坏、删除、导出第三方数据。
- 不要利用漏洞访问非授权系统或数据。

## 安全公告

修复发布后，安全公告建议包含：

- 漏洞摘要。
- 影响版本。
- 修复版本。
- 升级步骤。
- 临时缓解方式。
- 是否需要轮换密钥或强制用户重新登录。
- 致谢信息。

# 贡献指南

SmartAdmin 欢迎围绕通用后台能力的贡献。贡献目标不是堆功能，而是让权限、数据范围、租户隔离、日志审计、上传、发布升级和前端体验保持一致、可测试、可维护。

## 贡献类型

- 修复可复现缺陷。
- 补齐测试或文档。
- 优化权限、日志、上传、发布等核心链路。
- 新增通用后台模块或前端体验改进。

## 不建议的贡献

- 只适合单个业务项目的垂直业务功能。
- 绕过 Core 分层的新框架层。
- 无测试、无文档、无验证说明的大范围重构。
- 复用无关权限码来绕过授权设计。
- 记录明文密码、Token、Secret、Key 或上传签名。
- 依赖生产执行迁移文件逐条升级，绕过 release 安装包与运行备份机制。

## 提交流程

1. 基于最新主分支创建功能分支。
2. 阅读 [开发指南](../开发指南/README.md) 和相关功能说明。
3. 按项目分层完成代码和文档变更。
4. 运行与改动范围匹配的验证命令。
5. 提交 PR 前确认未包含 `runtime`、`vendor`、`build`、`public`、`web/dist`、前端构建压缩包、本地 `.env` 和 IDE 文件等本地/发布产物。
6. 提交 PR，说明背景、方案、影响和验证结果。

## 分支和提交

建议分支命名：

| 类型 | 示例 |
|------|------|
| 文档 | `docs/improve-open-source-guide` |
| 修复 | `fix/user-scope-export` |
| 功能 | `feat/file-upload-policy` |
| 发布 | `release/snapshot-dry-run` |

提交信息建议使用中文模块前缀：

```text
文档: 补充发布升级说明
权限: 修复用户导出数据范围
上传: 完善 OSS 配置脱敏
前端: 优化角色授权页面
```

## 验证要求

| 改动范围 | 建议命令 |
|----------|----------|
| 纯文档 | `composer docs:check` |
| 后端业务 | `composer analyse`、`composer test` |
| 前端页面/API | `composer web:build` |
| 发布升级 | `composer release:check` |
| 菜单/权限 | `./bin/smart xadmin:menu:sync --dry-run --json`、`./bin/smart xadmin:node:sync --dry-run --json` |

## 代码质量要求

后端贡献：

- Controller 只处理路由、参数、权限注解、日志注解和标准响应。
- Service 负责业务规则、事务、唯一性校验和跨 Mapper 编排。
- Mapper 负责查询、分页、数据范围、软删和列表后处理。
- Model 只负责表映射、fillable、hidden、关联、转换器和审计规则。
- 写操作必须有 `#[Logger]`，敏感字段必须脱敏。
- 有租户隔离需求的表必须包含 `tenant_id` 并接入通用租户范围。

前端贡献：

- API service、页面、表单、表格和按钮权限要与后端接口一致。
- 列表页需要搜索、分页、状态、回收站、导出等与后端能力匹配的入口。
- 高风险操作需要确认提示。
- 枚举、权限码和表单必填字段要与后端保持一致。

文档贡献：

- 新增接口必须补接口参考。
- 新增模块必须补用户教程、系统功能或开发指南。
- 修改部署、发布、上传、日志、租户、权限流程必须补部署运维或架构说明。
- 文档链接必须通过 `composer docs:check`。

## 文档同步要求

以下变更必须同步文档：

- 新增或修改接口、权限码、菜单、按钮权限。
- 修改登录、租户、数据范围、日志、上传、发布升级流程。
- 修改前端启动、构建、部署或运行时配置方式。
- 新增模块、命令、配置项或迁移策略。

## Review 关注点

维护者会优先检查：

- 是否保持现有分层，不把 SQL 或复杂业务放进 Controller。
- 是否覆盖权限、数据范围、租户和日志边界。
- 是否避免明文敏感数据进入日志、响应、导出和文档。
- 是否有匹配的测试或明确说明为什么不需要测试。
- 是否同步前端权限码、菜单、API service 和页面。
- 是否更新 docs。
- 是否引入与项目目标无关的大型依赖或重复框架层。

## 安全问题

安全漏洞不要直接提交公开 PR 公开细节。请先阅读 [安全与漏洞披露](./安全与漏洞披露.md)，通过安全披露渠道提供影响范围、复现步骤、版本信息和建议修复方向。

# 会员机制

SmartAdmin 会员机制用于支持项目长期维护、插件扩展、学习交流、实时沟通和商业服务。会员不是 Apache-2.0 开源协议的附加限制，不影响开源代码本身的使用、修改、再分发和商业使用权利。

## 会员定位

| 对象 | 说明 |
|------|------|
| 非会员用户 | 可按 Apache-2.0 协议使用 SmartAdmin 开源代码，并阅读公开文档 |
| 会员用户 | 在公开能力基础上，获得会员权益、会员群交流、会员授权插件交付和更及时的生态信息 |
| 商业用户 | 可通过会员或有偿服务获得更稳定的沟通、排障、定制和学习支持 |

公开文档站会展示会员授权插件的接口与使用说明，方便提前评估能力和联调边界；但文档可见不代表插件源码或 ZIP 已随开源版分发，实际安装使用仍以会员授权或内部交付为准。

## 会员权益

- 进入会员微信群，参与实时互动、技术沟通和学习交流。
- 更快获取 SmartAdmin 生态信息、更新说明和实践经验。
- 围绕安装部署、二次开发、插件使用、项目落地进行更直接的答疑。
- 在需要定制开发、问题排查或专项支持时，可优先建立沟通。
- 享受后续会员插件、会员服务或折扣权益。

## 开通方式

当前会员开通以添加微信沟通为主。添加好友后说明使用场景、项目来源和开通需求，由维护者确认会员信息。

| 项目 | 内容 |
|------|------|
| 微信号 | `ThinkAdmin_TOP` |
| 添加说明 | 添加好友时请说明“开通会员”或“SmartAdmin 交流学习” |
| 会员费用 | 当前会员费用、有效期和权益以维护者沟通确认为准 |
| 授权管理 | 会员授权、历史交易和后续服务信息由会员授权系统或维护者统一确认 |

## 微信群规则

- 入群需要验证基本信息，请说明项目来源、使用场景和加群原因。
- 群内以技术交流、学习沟通、生态更新和项目实践为主。
- 不发布广告、无关推广、违法内容或影响技术交流的信息。
- 添加微信后建议不要删除，避免会员记录、沟通记录或后续服务信息丢失。
- 入群、会员、插件和服务规则以维护者实际确认的权益范围为准。

## 与开源协议的关系

SmartAdmin 开源代码采用 Apache-2.0 协议。会员机制不改变开源代码本身的使用、修改、再分发和商业使用权利；会员权益主要用于支持项目长期维护，并提供更实时的互动沟通、学习支持和生态服务。

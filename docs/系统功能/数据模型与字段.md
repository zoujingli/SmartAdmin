# 数据模型与字段

本页按当前 Model 实现整理核心数据表字段语义。字段以 `$fillable`、访问器、关联关系和审计规则为准，适合二次开发建模前查阅。

## 用户模型

模型：`SystemUser`
表：系统用户表
软删除：是
敏感字段：`password`

| 字段 | 说明 |
|------|------|
| `tenant_id` | 租户 ID，普通业务查询依赖租户上下文隔离 |
| `username` | 登录账号，必须唯一 |
| `nickname` | 用户昵称 |
| `phone`、`email` | 联系方式 |
| `password` | 写入时自动 `password_hash`，隐藏返回 |
| `avatar`、`signed` | 头像和个性签名 |
| `status` | `1` 启用，`0` 禁用 |
| `login_ip`、`login_time` | 最近登录信息 |
| `extra` | JSON 扩展字段，保存界面偏好等数据 |

关联：用户与角色、部门、岗位均为多对多。用户权限由角色聚合，超级管理员可短路权限校验。

## 角色模型

模型：`SystemRole`
软删除：是

| 字段 | 说明 |
|------|------|
| `name` | 角色名称 |
| `scope` | 数据范围，支持全部、本部门、部门及下级、仅本人 |
| `sort` | 排序权重 |
| `status` | 状态 |
| `remark` | 备注 |

关联：角色与用户、权限节点多对多。角色身份使用主键 ID，授权节点通过 `system_role_node` 维护。

## 菜单模型

模型：`SystemMenu`
软删除：是

| 字段 | 说明 |
|------|------|
| `pid`、`level` | 菜单树父级和层级 |
| `name`、`code` | 菜单名称和权限编码 |
| `type` | 菜单类型，区分目录、菜单、按钮等 |
| `route`、`component`、`redirect` | 前端路由、组件和重定向 |
| `link`、`iframe_src` | 外链和 iframe 页面 |
| `hide_in_menu`、`hide_in_breadcrumb`、`hide_in_tab` | 前端显示控制 |
| `keep_alive`、`affix_tab` | 页面缓存和固定标签页 |
| `sort`、`status` | 排序和状态 |

菜单 code 会参与权限节点同步，前端按钮权限必须与后端 `#[Auth]` code 对齐。

## 部门与岗位

部门模型：`SystemDept`
岗位模型：`SystemPost`

部门字段包括 `pid`、`name`、`phone`、`level`、`leader`、`sort`、`status`、`remark`。部门是树结构，和用户多对多。

岗位字段包括 `code`、`name`、`sort`、`status`、`remark`。岗位用于描述用户职责，和用户多对多。

## 文件模型

模型：`SystemFile`
软删除：是

| 字段 | 说明 |
|------|------|
| `scene` | 上传场景 |
| `driver` | 存储驱动，例如 local、oss、cos、qiniu、alist |
| `url` | 访问 URL |
| `hash` | 文件 hash，用于秒传和去重 |
| `suffix`、`mime_type` | 后缀和 MIME |
| `storage_mode` | 存储模式 |
| `origin_name`、`object_name` | 原始文件名和对象名 |
| `storage_path` | 存储路径 |
| `size_byte`、`size_info` | 文件大小 |

## 日志与公告

操作日志模型 `SystemLogsAction` 保存 `username`、`method`、`router`、`name`、`remark`、`ip`、`ip_location`、`os`、`browser`、`request_data`、`response_code`、`response_data` 等字段。

变更日志模型 `SystemLogsChange` 通过 `action_id` 关联操作日志，保存 `model`、`table_name`、`record_id`、`record_label`、`event`、`change_values`、`change_remark` 等字段。

公告模型 `SystemNotice` 保存 `title`、`content`、`level`、`status`、`published_at`、`expired_at`、`link`。公告接收关系由 `SystemNoticeUser` 维护，包含已读和归档状态。

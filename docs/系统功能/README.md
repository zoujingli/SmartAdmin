# 系统功能

本章按实现模块解释 SmartAdmin 的能力边界。后端主体在 `plugin/*`，Web 作为通用前端壳和共享运行库；插件页面由各插件通过 `plugin.json` 和 `plugin.view_root` 显式声明后自管理，并在前端构建时参与映射。当前插件机制不是运行时远程插件加载；源码/CI 可使用 SmartAdminLibrary 提供的 `xadmin:plugin:*` 做 ZIP 打包、安装、移除和备份恢复，备份默认只含代码，显式 `--with-data` 才包含插件自有表数据，发布二进制只运行已打包插件且不注册这些源码命令。

## 阅读目标

读完本章后，你应该能回答：

- 系统内置哪些通用后台能力。
- 每个能力由哪些后端模块和前端页面实现。
- 新模块接入时应该复用哪些能力。
- 权限、数据范围、租户、日志、上传和发布之间如何协作。

## 模块分层

| 模块 | 职责 |
|------|------|
| `zoujingli/smart-admin-library` | 基类、注解、异常、日志、数据范围、租户上下文、release 安装包、站点中间件、源码插件管理命令 |
| `plugin/System` | 认证、用户、角色、菜单、部门、岗位、租户、文件、日志、公告、系统数据 |
| `zoujingli/smart-admin-builder` | Phar 打包和配置 AST 改写 |
| `web/apps/web-antd` | 通用管理端壳、布局、公共组件、共享依赖、插件扫描和运行时配置 |

## 能力概览

| 能力 | 作用 | 关键页面/接口 |
|------|------|---------------|
| 认证和个人资料 | 登录、刷新 Token、个人资料、密码、偏好 | `/system/auth/*` |
| RBAC 权限 | 用户、角色、菜单、按钮、权限节点 | `/system/user`、`/system/role`、`/system/menu` |
| 数据范围 | 控制用户可见和可操作的数据 | 角色 scope、Mapper 查询保护 |
| 组织模型 | 部门树、岗位、用户归属 | `/system/dept`、`/system/post` |
| 文件上传 | 上传配置、驱动、分片、下载、去重 | `/system/file` |
| 日志审计 | 请求日志、操作日志、字段变更、关联回看 | `/system/logs/action`、`/system/logs/change` |
| 公告通知 | 管理端公告、用户收件箱、已读归档 | `/system/notice` |
| 多租户 | 租户资料、状态、租户上下文 | `/system/tenant` |
| 发布构建 | release 安装包、运行备份、dry-run、Phar 打包 | `xadmin:release:*`、`xadmin:build:phar` |

## 二次开发复用建议

| 需求 | 优先复用 |
|------|----------|
| 新增后台资源 | Core CRUD 分层、`plugin.json` 菜单/按钮清单、权限节点、日志注解 |
| 新增附件字段 | 文件上传接口和文件记录 |
| 新增审计要求 | `#[Logger]` 和 Model `$logRules` |
| 新增组织隔离 | 部门数据范围和角色 scope |
| 新增租户业务 | `TenantContext` 和 `tenant_id` 模型范围 |
| 新增发布种子数据 | release 安装包和 restore dry-run |

# 文档系统部署

本项目文档基于 docsify 构建，源码统一放在 `docs/` 目录。docsify 在浏览器中直接加载 Markdown 文件，不需要构建步骤，部署时只需要把 `docs/` 目录作为静态资源目录发布。

## 目录约定

- `docs/index.html` 是文档系统入口，负责加载 docsify、搜索、代码复制、Mermaid 图表和在线调试插件。
- 默认路由直接加载 `快速开始/README.md` 正文，避免先展示独立首页或封面页。
- `docs/README.md` 仅作为文档总览存档，不再作为站点默认入口。
- `docs/_sidebar.md` 是左侧目录，跨章节入口必须使用 `/章节/页面` 这种 docsify 根路径写法，新增页面后必须同步维护。
- 顶部模块导航由 `docs/assets/devapi-module-nav.js` 根据 `_sidebar.md` 的一级分组生成，不再维护独立导航文件。
- `docs/_404.md` 是未匹配页面的兜底说明，避免无效链接直接展示空白内容。
- `docs/.nojekyll` 用于 GitHub Pages 等静态托管场景，避免以下划线开头的 docsify 文件被 Jekyll 忽略。
- `docs/assets/devapi-docs.css`、`devapi-scale.js`、`devapi-module-nav.js`、`devapi-sidebar-state.js` 和 `devapi-mermaid.js` 负责统一视觉、DPI 缩放、顶部模块导航、左侧目录状态和 Mermaid 图表渲染。
- `docs/assets/api-tester.css`、`api-tester.js` 负责接口参考页的在线调试弹窗。

## 视觉资源

当前文档站参考 DevAPI 工作台风格：

| 资源 | 用途 |
|---|---|
| `assets/devapi-docs.css` | 统一背景、正文卡片、标题、表格、代码块、搜索框和左侧菜单样式 |
| `assets/devapi-scale.js` | 根据浏览器 DPI 设置 `--devapi-ui-scale`，保证高分屏下字号和间距稳定 |
| `assets/devapi-module-nav.js` | 从 `_sidebar.md` 一级分组生成顶部模块导航，并提供当前 Markdown 下载入口 |
| `assets/devapi-sidebar-state.js` | 为 docsify 左侧目录补充折叠/展开按钮，并按站点路径记忆滚动和展开状态 |
| `assets/devapi-mermaid.js` | 将 Markdown 中的 `mermaid` 代码块渲染为图表，支持 `flowchart LR`、`flowchart TB` 等语法 |
| `assets/api-tester.css` | 统一在线调试入口、弹窗、表单和响应区样式 |
| `assets/api-tester.js` | 渲染 `api-test` 代码块，处理 Token、密码加密、请求发送和响应展示 |

修改这些资源后必须同步执行 `composer docs:check`，确保 `docs/index.html` 仍加载完整资源。

## 部署到 `/docs`

将仓库根目录作为站点根目录时，保留 `docs/` 目录即可通过 `/docs/` 访问文档。

```nginx
location /docs/ {
    root /path/to/SmartAdmin;
    index index.html;
    try_files $uri $uri/ /docs/index.html;
}
```

访问地址：

```text
https://example.com/docs/
https://example.com/docs/#/快速开始/README
```

## 部署到 `/`

如果文档需要作为独立站点部署到根路径，将 `docs/` 目录内容发布为静态站点根目录。

```nginx
server {
    listen 80;
    server_name docs.example.com;
    root /path/to/SmartAdmin/docs;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

访问地址：

```text
https://docs.example.com/
https://docs.example.com/#/快速开始/README
```

## 链接规则

- 站内 Markdown 链接优先写相对路径，例如 `[本地启动](../快速开始/本地启动.md)`。
- 侧边栏是全站导航，必须写 `/章节/页面`，否则从子目录页面点击跨章节链接时会被拼接到当前目录下。
- 导航等直接跳 docsify 路由的位置统一写 `<a href="./#/章节/页面">`，不要写 `/#/章节/页面` 或 Markdown 格式的 `[#/章节/页面](#/章节/页面)`。
- 不要在文档中写死域名、部署目录或站点根路径，避免 `/docs` 和 `/` 两种部署方式行为不一致。

## 本地预览

可以用任意静态服务预览 `docs/` 目录。

```bash
npx docsify-cli serve docs
```

也可以使用 PHP 内置静态服务。

```bash
php -S 127.0.0.1:3000 -t docs
```

本地访问：

```text
http://127.0.0.1:3000/
```

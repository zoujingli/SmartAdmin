## 样式基线

当前 docs 参考 DevAPI 工作台的阅读体验，统一采用：

| 项目 | 当前实现 |
|---|---|
| 默认入口 | 站点根路径直接展示 `快速开始/README.md` 正文 |
| 顶部导航 | `devapi-module-nav.js` 从侧边栏一级分组生成模块导航，并支持下载当前 Markdown |
| 侧边栏 | 中文业务分组 + 可折叠树形目录 + 展开/收起工具 + 滚动位置记忆 |
| 视觉资源 | `assets/devapi-docs.css`、`devapi-scale.js`、`devapi-module-nav.js`、`devapi-sidebar-state.js`、`devapi-mermaid.js` |
| 在线调试 | `assets/api-tester.css`、`api-tester.js` |
| 翻页 | `docsify-pagination` 提供上一章节/下一章节卡片 |

调整样式、脚本或导航结构时，必须同步 [文档系统部署](./文档系统部署.md) 和 `xadmin:docs:check` 校验项。

## 维护原则

- 代码变更先落地，再同步文档
- 规范变化更新对应主题文档，接口变化同步接口参考，版本变化更新日志
- 同一主题只保留一个主文档，避免重复与冲突
- 补充专题必须服务主线文档，避免形成第二套口径
- 源码模式、CI 模式、发布二进制模式必须分开描述，不能把开发迁移、插件安装或构建命令写成生产运行步骤
- 会员授权插件可以公开能力和接口文档，但必须标清源码、ZIP 和安装使用不随开源版分发
- 接口文档必须来自 Controller、继承 CRUD、Service、Mapper、Model 和前端表单，不能只按页面名称或历史习惯补路径

## 口径收口清单

每次批量更新 docs 后，至少复查以下内容：

| 口径 | 必须保持一致 |
|------|--------------|
| 命令入口 | 统一使用 `./bin/smart.php`；生产必须显式 `start`；旧入口不再作为维护口径 |
| 插件机制 | 本地 Composer path 包 + Provider + `plugin.json` + 菜单/节点同步 + Web 编译期扫描 |
| 插件命令 | `xadmin:plugin:*` 仅源码/CI 可用，发布 Phar/SFX 不注册动态插件命令 |
| 发布升级 | 正式安装升级使用 release 安装包与 restore dry-run，不在生产二进制执行 migrate |
| 接口示例 | 请求/响应都用 JSONC，保留 `code/info/data/path`，并带中文注释和 `api-test` 调试块 |
| 授权边界 | “文档可见”不等于“开源版包含源码或 ZIP” |

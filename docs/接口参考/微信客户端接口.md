# 微信客户端接口

本页整理 `plugin/WechatClient` 提供的公众号/小程序接口账号、粉丝、菜单、自动回复、素材、文章、微信支付和回调接口。

- `plugin/WechatClient/plugin.json`

新增或调整页面时维护插件自己的 `plugin.json` 与 `plugin.view_root`，Web 只负责通用运行库和公共组件加载；未配置资源根的插件页面不会参与打包映射。

## 接口内容

本页覆盖 WechatClient 插件面向后台和微信服务器暴露的 HTTP 接口，包含三类协议：

| 协议类型 | 覆盖范围 | 调试重点 |
|---|---|---|
| 后台权限接口 | 账号、粉丝、菜单、回复、素材、文章、支付商户、订单、退款 | 使用后台 Bearer Token，并确认角色具备对应权限码 |
| 微信消息回调 | `/wechat-client/api/push/{appid}` | 微信服务器携带 `signature` 或 `msg_signature`，服务端完成验签/解密 |
| 微信支付回调 | `/wechat-client/api/payment/notify/*` | 必须传微信支付 APIv3 原始 body 和签名请求头，SDK 完成验签与资源解密 |

## 调试说明

- 后台接口统一使用 `Authorization: Bearer <token>`，响应壳为 `code/info/data/path`。
- 写接口会记录操作日志，`appsecret`、`token`、`encodingaeskey`、APIv3 Key、证书私钥和平台公钥等敏感字段会加密保存或脱敏。
- 公众号直连接口使用新版 `wechat-developer` 的 `wechatPlatform` 通道；小程序直连接口使用 `wechatWxapp` 通道；开放平台模式通过 WechatService JSON-RPC 网关代调用。
- 菜单发布按微信官方限制校验：一级菜单最多 3 个，二级菜单最多 5 个，名称长度按一级 8 个中文字符、二级 16 个中文字符控制。
- 支付金额统一使用“分”；主动查单/查退只作为通知丢失补偿，本地已终态时不重复访问微信。

## 对接说明

1. 先维护接口账号。直连模式填写 AppID、AppSecret、Token、EncodingAESKey；开放平台模式填写 JSON-RPC 网关地址、Key 和 Secret。
2. 公众号回调地址配置为 `/wechat-client/api/push/{appid}`，安全模式会自动解密，产生被动回复时会重新加密。
3. 菜单、素材、文章和粉丝同步依赖微信官方接口权限，接口异常会以业务 `500` 返回。
4. 启用支付前维护支付商户，并在微信支付后台配置订单与退款通知地址。
5. 业务模块优先调用 `WechatClientPaymentService`，不要绕过 Service 直接写支付单或退款单。

## 使用边界

- 权限接口需要登录态和对应按钮/菜单权限码，权限码必须与 `plugin.json` 的按钮声明和 Controller `#[Auth]` 保持一致。
- 公开回调用于微信服务器或内部网关访问，不应依赖前端权限；签名、Token、密钥和回调数据必须在 Service 层校验并脱敏记录。
- 涉及 `appsecret`、`token`、`encodingaeskey`、证书私钥、`client_secret` 等敏感字段时，日志和接口响应都不得明文暴露。
- 支付查单、查退是通知丢失时的补偿入口；本地记录已处于终态时，服务会优先返回本地状态，避免重复访问微信接口。


## 常用对象字段

### 接口账号

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | integer | 返回时是 | 接口账号 ID |
| `tenant_id` | integer | 是 | 租户 ID，新建时默认当前租户 |
| `appid` | string | 是 | 微信公众号或小程序 AppID，全局唯一 |
| `name` | string | 是 | 后台展示名称 |
| `account_type` | string | 否 | `official_account` 公众号，`mini_program` 小程序 |
| `service_mode` | integer | 否 | `0` 直连，`1` 开放平台网关模式 |
| `appsecret` | string | 直连接口调用需要 | 加密保存；编辑时传 `******` 表示保持原密文 |
| `token` | string | 回调验签需要 | 加密保存；明文模式和安全模式都依赖 Token |
| `encodingaeskey` | string | 安全模式需要 | 43 位 EncodingAESKey，加密保存 |
| `extra.gateway_url` | string | 开放平台模式需要 | JSON-RPC 网关完整 URL，不允许 query、fragment 或认证信息 |
| `extra.gateway_client_key` | string | 开放平台模式需要 | 网关调用 Key |
| `extra.gateway_client_secret` | string | 开放平台模式需要 | 网关调用 Secret，加密保存 |

### 支付商户

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `appid` | string | 是 | 支付 AppID，通常为公众号或小程序 AppID |
| `mch_id` | string | 是 | 微信支付商户号 |
| `api_v3_key` | string | 是 | 微信支付 APIv3 Key，32 字节字符串，加密保存 |
| `merchant_serial` | string | 是 | 商户 API 证书序列号，加密保存 |
| `merchant_private_key` | string | 是 | 商户私钥 PEM，加密保存 |
| `platform_public_key` | string | 回调验签需要 | 微信支付平台公钥或证书 PEM，加密保存 |
| `platform_serial` | string | 建议填写 | 微信支付平台公钥或证书序列号，用于回调头序列号校验 |

## 错误码

| code | 常见消息 | 说明 |
|---|---|---|
| `401` | Token 相关错误 | 后台权限接口未登录或 Token 失效 |
| `403` | 无操作权限 | 当前账号缺少对应权限码 |
| `500` | 微信接口账号不可用 | 账号不存在、禁用或回调 AppID 未匹配 |
| `500` | 开放平台网关配置不完整 | 网关模式缺少 URL、Key 或 Secret |
| `500` | 微信回调签名验证失败 | Token、timestamp、nonce、signature 或密文不匹配 |
| `500` | 微信支付回调验签失败 | 微信支付平台公钥、序列号、请求头或原始 body 不匹配 |
| `500` | 支付订单不存在 / 退款记录不存在 | 通知中的支付号或退款号没有匹配本地记录 |

# 角色菜单接口

## 角色接口

控制器：`plugin/System/src/Controller/RoleController.php`
前缀：`/system/role`

| 方法 | 路径 | 认证 | 权限码 | 日志 | 用途 |
|------|------|------|--------|------|------|
| GET | `/system/role/index` | 权限 | `system.role.index` | 否 | 角色列表 |
| GET | `/system/role/recycle` | 权限 | `system.role.index` | 否 | 角色回收站 |
| GET | `/system/role/info/{id}` | 权限 | `system.role.index` | 否 | 角色详情 |
| POST | `/system/role/create` | 权限 | 默认角色创建权限 | 新增系统角色 | 新增角色 |
| PUT | `/system/role/update/{id}` | 权限 | 默认角色编辑权限 | 编辑系统角色 | 编辑角色 |
| DELETE | `/system/role/delete/{ids}` | 权限 | 默认角色删除权限 | 删除系统角色 | 删除到回收站 |
| DELETE | `/system/role/real-delete/{ids}` | 权限 | 默认角色彻底删除权限 | 彻底删除系统角色 | 彻底删除 |
| PUT | `/system/role/recovery/{ids}` | 权限 | 默认角色恢复权限 | 恢复系统角色 | 恢复 |

| GET | `/system/role/nodes/{id}` | 权限 | `system.role.assign` | 否 | 获取角色权限 |
| GET | `/system/role/users/{id}` | 权限 | `system.role.index` | 否 | 获取角色用户 |
| GET | `/system/role/options` | 权限 | `system.role.index` | 否 | 获取角色选项 |
| GET | `/system/role/permission-tree` | 权限 | `system.role.assign` | 否 | 获取角色授权菜单 |
| GET | `/system/role/statistics` | 权限 | `system.role.index` | 否 | 角色统计 |

### 角色查询与写入

| 场景 | 参数/字段 | 说明 |
|------|-----------|------|
| 列表查询 | `name`、`code`、`status`、`page`、`pageSize` | 常见筛选条件，实际以 Mapper 白名单为准 |
| 创建/编辑 | `name`、`code`、`scope`、`sort`、`status`、`remark` | 不包含权限节点 |
| 状态切换 | `status` | 启用或禁用角色 |
| 角色选项 | 无或关键字 | 用于用户分配角色下拉选项 |
| 角色用户 | `id` 路径参数 | 查看当前角色关联的用户 |

### 数据范围

| 值 | 含义 | 说明 |
|----|------|------|
| `1` | 全部数据 | 最高数据范围，谨慎授予 |
| `2` | 自定义数据 | 需要结合自定义部门或节点配置 |
| `3` | 本部门数据 | 只看用户所属部门 |
| `4` | 本部门及下级数据 | 适合部门负责人 |
| `5` | 仅本人数据 | 适合普通操作员 |

### 权限节点分配

`GET /system/role/permission-tree` 返回当前操作者可授权的菜单/按钮树。`PUT /system/role/nodes/{id}` 保存角色节点，常见请求体如下：

```json
{
  "node_ids": [1, 2, 3]
}
```
保存后用户需要重新登录或刷新权限缓存，前端才能拿到新的菜单和按钮权限。服务端会记录授权操作日志。

### 菜单查询与写入

| 场景 | 参数/字段 | 说明 |
|------|-----------|------|
| 列表/树 | `name`、`code`、`type`、`status` | 返回树形结构或可排序列表 |
| 创建/编辑 | `pid`、`name`、`code`、`icon`、`type`、`route`、`component`、`redirect`、`sort`、`status` | 菜单核心字段 |
| 路由控制 | `link`、`iframe_src`、`hide_in_menu`、`hide_in_breadcrumb`、`hide_in_tab`、`keep_alive`、`affix_tab` | 前端路由和页签行为 |
| 状态切换 | `status` | 控制菜单或按钮是否可用 |
| 排序 | `sort` | 同级菜单排序 |

### 菜单类型和权限码

- 目录用于菜单分组，通常不配置具体页面组件。
- 菜单用于页面路由，`route` 和 `component` 必须能被前端路由系统识别。
- 按钮用于操作权限，`code` 必须与后端 `#[Auth]` code 对齐，例如 `system.user.update`。
- `node-options` 只返回后端注解同步的权限节点建议，用于菜单新增/编辑时提示“节点代码 + 中文描述”，不限制手动输入。
- `user` 和 `permissions` 接口属于登录态接口，用于前端渲染当前用户菜单和按钮权限。

## 同步建议

新增后端接口或前端按钮后，建议按以下顺序处理：

1. 在 Controller 上补充 `#[Auth]` 权限码。
2. 在菜单管理中新增按钮节点或调整菜单 code。
3. 执行菜单/节点同步命令。
4. 在角色授权中勾选新节点。
5. 使用低权限用户验证菜单可见性、按钮可见性和接口返回码。

### 上传通道配置

`upload-config` 用于读取和保存服务端上传配置。保存配置时敏感字段会按点路径脱敏，例如 `drivers.oss.access_secret`、`drivers.cos.secret_key`。

| 配置块 | 说明 |
|--------|------|
| `default_driver` | 默认上传驱动 |
| `max_size` | 单文件大小限制 |
| `chunk_size` | 分片大小 |
| `allowed_extensions` | 允许后缀 |
| `allowed_mime_types` | 允许 MIME |
| `scenes` | 业务上传场景配置 |
| `drivers.local` | 本地存储配置 |
| `drivers.alist` | Alist 网关配置 |
| `drivers.qiniu` | 七牛云配置 |
| `drivers.oss` | 阿里云 OSS 配置 |
| `drivers.cos` | 腾讯云 COS 配置 |

### 中转上传

小文件或需要服务端统一校验的文件可使用中转上传：

```http
POST /system/file/upload/relay
Content-Type: multipart/form-data
Authorization: Bearer <token>
```

常见表单字段：

| 字段 | 说明 |
|------|------|
| `file` | 上传文件 |
| `scene` | 上传场景 |
| `driver` | 可选，指定驱动 |
| `remark` | 可选备注 |

### 分片上传

大文件建议使用分片流程：

1. `prepare` 创建上传会话，返回上传会话 ID、分片大小、驱动和上传策略。
2. `relay-chunk` 上传服务端中转分片。
3. 远端直传场景通过 `part-sign` 获取分片签名，由前端上传到对象存储。
4. `complete` 合并或确认文件，服务端写入文件记录。
5. 上传失败或用户取消时调用 `abort` 清理会话。

常见分片字段：

| 字段 | 说明 |
|------|------|
| `upload_id` | 上传会话 ID |
| `chunk_index` | 分片序号 |
| `chunk_total` | 总分片数 |
| `chunk_hash` | 分片哈希 |
| `file_hash` | 整体文件哈希 |
| `file_name` | 原始文件名 |
| `file_size` | 文件大小 |
| `scene` | 上传场景 |

### 权限与安全

- 文件管理接口需要菜单权限；上传流程只要求登录，但仍必须校验场景、大小、后缀、MIME 和驱动配置。
- 下载接口需要 `system.file.index` 权限，避免通过文件 ID 绕过管理权限。
- 签名下载接口不要求登录，访问权限由 `expires`、`attname`、`signature` 短期签名参数控制。
- 对象存储密钥只能保存在服务端配置中，前端只能拿到临时签名或必要的公开访问地址。
- 文件 URL 进入业务表后，删除文件记录前必须确认业务引用关系。

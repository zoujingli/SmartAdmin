# 日志公告接口

## 操作日志接口

控制器：`plugin/System/src/Controller/LogsActionController.php`
前缀：`/system/logs/action`

| 方法 | 路径 | 认证 | 权限码 | 日志 | 用途 |
|------|------|------|--------|------|------|
| GET | `/system/logs/action/index` | 权限 | `system.logs.action.index` | 否 | 操作日志列表 |
| GET | `/system/logs/action/recycle` | 权限 | `system.logs.action.index` | 否 | 操作日志回收站 |
| GET | `/system/logs/action/info/{id}` | 权限 | `system.logs.action.index` | 否 | 操作日志详情 |
| DELETE | `/system/logs/action/delete/{ids}` | 权限 | `system.logs.action.delete` | 删除操作日志 | 删除到回收站 |
| DELETE | `/system/logs/action/real-delete/{ids}` | 权限 | `system.logs.action.real-delete` | 彻底删除操作日志 | 彻底删除 |
| PUT | `/system/logs/action/recovery/{ids}` | 权限 | `system.logs.action.recovery` | 恢复操作日志 | 恢复 |
| DELETE/POST | `/system/logs/action/clear` | 权限 | `system.logs.action.clear` | 清空操作日志 | 清空日志 |
| GET | `/system/logs/action/statistics` | 权限 | `system.logs.action.index` | 否 | 操作日志统计 |
| GET | `/system/logs/action/analysis` | 权限 | `system.logs.action.index` | 否 | 操作日志分析报告 |
| GET | `/system/logs/action/metrics` | 权限 | `system.logs.action.index` | 否 | 操作日志近实时指标 |

### 操作日志查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `page` / `pageSize` | number | 分页参数 |
| `keyword` | string | 综合关键字，匹配操作人、路由、操作名、IP、摘要 |
| `username` | string | 操作人 |
| `method` | string | HTTP 方法 |
| `router` | string | 路由关键字 |
| `name` | string | 操作名称 |
| `response_code` | string | 响应业务码 |
| `ip` | string | 请求 IP |
| `created_at` | string[] | 创建时间范围 |

## 变更日志接口

控制器：`plugin/System/src/Controller/LogsChangeController.php`
前缀：`/system/logs/change`

| 方法 | 路径 | 认证 | 权限码 | 日志 | 用途 |
|------|------|------|--------|------|------|
| GET | `/system/logs/change/index` | 权限 | `system.logs.change.index` | 否 | 变更日志列表 |
| GET | `/system/logs/change/info/{id}` | 权限 | `system.logs.change.index` | 否 | 变更日志详情 |
| GET | `/system/logs/change/statistics` | 权限 | `system.logs.change.index` | 否 | 变更日志统计 |
| GET | `/system/logs/change/action/{id}` | 权限 | `system.logs.action.index` | 否 | 指定操作日志关联的变更明细 |

### 变更日志查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `page` / `pageSize` | number | 分页参数 |
| `keyword` | string | 综合关键字，匹配操作人、模型、业务记录和变更摘要 |
| `tenant_id` | number | 租户 ID |
| `action_id` | number | 关联操作日志 ID |
| `model` | string | 模型短名 |
| `table_name` | string | 业务表名 |
| `event` | string | `created`、`updated`、`deleted`、`force_deleted`、`restored` |
| `created_at` | string[] | 创建时间范围 |

## 日志审计约束

- 操作日志写入 `system_logs_action`，只记录接口行为、请求响应摘要、客户端信息和操作摘要。
- 变更日志写入 `system_logs_change`，通过 `action_id` 关联操作日志，一条业务对象变化对应一条变更日志。
- 业务模型的新增、编辑、软删除、物理删除、恢复通过模型事件记录变更；关系授权、配置数组等模型事件不稳定场景由 Service 显式追加变更。
- 删除、清空、彻底删除日志自身只记录操作日志，并同步处理关联变更日志，不生成递归变更日志。

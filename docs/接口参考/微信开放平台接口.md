## 接口内容

本页覆盖 WechatService 插件的开放平台配置、授权账号、网关凭据、微信公开回调和内部 JSON-RPC 网关接口。它通常与 WechatClient 配合使用：WechatService 负责第三方平台授权和代调用，WechatClient 负责租户侧账号、内容互动和支付业务。

| 协议类型 | 覆盖范围 | 调试重点 |
|---|---|---|
| 后台权限接口 | 平台配置、授权账号、网关凭据 | 使用后台 Bearer Token，并确认权限码 |
| 微信开放平台回调 | 授权回调、Ticket 回调、授权账号消息回调 | 微信服务器携带签名参数，服务端完成验签/解密和事件分发 |
| 内部 JSON-RPC | `/wechat-service/api/rpc/jsonrpc` | 使用网关 Token，按 JSON-RPC 2.0 发送白名单方法 |

## 调试说明

- 后台接口响应壳为 `code/info/data/path`；公开回调和 JSON-RPC 按各自协议返回原始文本或 JSON-RPC 结构。
- 平台配置中的 `component_appsecret`、`component_token`、`component_encodingaeskey`、`component_verify_ticket` 和 `component_access_token` 加密保存，详情接口只返回配置状态或掩码。
- `component_access_token`、`authorizer_access_token` 由新版 `wechat-developer` SDK 统一缓存和刷新，缓存键带平台与配置作用域。
- JSON-RPC 网关只允许相对微信 API path，不允许绝对 URL、内联 query 或 fragment，避免把代调用能力放大成 SSRF。

## 对接说明

1. 在微信开放平台创建第三方平台，获取 Component AppID、AppSecret、Token、EncodingAESKey。
2. 后台保存平台配置后，配置授权事件接收 URL：`/wechat-service/api/callback/ticket`。
3. 收到 `component_verify_ticket` 后，调用授权链接接口生成授权 URL 并完成租户绑定。
4. 授权账号消息与事件接收 URL 必须使用 `/wechat-service/api/callback/notify/$APPID$`，由微信开放平台替换为授权方 AppID。
5. 给 WechatClient 开放代调用时，先创建网关凭据，再把 JSON-RPC 地址、Key、Secret 写入 WechatClient 接口账号扩展配置。

## 使用边界

- 权限接口需要登录态和对应按钮/菜单权限码，权限码必须与 `plugin.json` 的按钮声明和 Controller `#[Auth]` 保持一致。
- 公开回调用于微信服务器或内部网关访问，不应依赖前端权限；签名、Token、密钥和回调数据必须在 Service 层校验并脱敏记录。
- 涉及 `appsecret`、`token`、`encodingaeskey`、证书私钥、`client_secret` 等敏感字段时，日志和接口响应都不得明文暴露。


## 常用对象字段

### 开放平台配置

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `component_appid` | string | 是 | 微信开放平台第三方平台 AppID |
| `component_appsecret` | string | 是 | 第三方平台 AppSecret，加密保存 |
| `component_token` | string | 是 | 回调 Token，用于明文验签和安全模式签名 |
| `component_encodingaeskey` | string | 是 | 第三方平台 EncodingAESKey，43 位字符串 |
| `component_verify_ticket` | string | 回调后写入 | 微信定时推送的 Ticket，加密保存，是换取 `component_access_token` 的前置凭据 |
| `status` | integer | 否 | `1` 启用，`0` 禁用 |

### 授权账号

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `authorizer_appid` | string | 是 | 授权公众号或小程序 AppID |
| `tenant_id` | integer | 是 | 授权绑定的租户 ID |
| `account_type` | string | 返回时是 | `official_account`、`subscription` 或 `mini_program` |
| `authorizer_refresh_token` | string | 授权后写入 | 授权方刷新令牌，加密保存 |
| `expires_at` | integer | 返回时是 | 授权方 access token 过期时间戳 |
| `permissions` | array | 返回时是 | 微信返回的授权权限集 |

### 网关凭据

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `client_key` | string | 是 | 调用方 Key，可由后端生成 |
| `client_secret` | string | 新建/轮换时返回 | 调用方 Secret，只返回一次，加密保存 |
| `authorizer_appid` | string | 是 | 允许代调用的授权方 AppID |
| `status` | integer | 否 | `1` 启用，`0` 禁用 |

## JSON-RPC 方法

| 方法 | 参数 | 返回 | 说明 |
|---|---|---|---|
| `wechat.platform.request` | `uri`、`http_method`、`query`、`payload` | 微信官方接口响应数组 | 代授权账号调用微信公众平台或小程序接口 |
| `authorizer.info` | 无 | 授权账号资料 | 查询当前网关 Token 绑定的授权账号 |

## 错误码

| code | 常见消息 | 说明 |
|---|---|---|
| `401` | Token 相关错误 | 后台权限接口未登录或 Token 失效 |
| `403` | 无操作权限 | 当前账号缺少对应权限码 |
| `500` | 微信开放平台配置未启用 | 未保存配置或配置被禁用 |
| `500` | 尚未收到开放平台 component_verify_ticket | 还不能生成授权链接或刷新 component token |
| `500` | 微信回调签名验证失败 | 回调 Token、签名参数或密文不匹配 |
| `500` | JSON-RPC 方法不支持 | 网关只开放白名单方法 |
| `500` | 微信接口路径无效 | 代调用 path 为空、绝对 URL、内联 query 或 fragment |

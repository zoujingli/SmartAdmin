# System 后台认证接口

控制器：`plugin/System/src/Controller/AuthController.php`
前缀：`/system/auth`


| 方法 | 路径 | 认证 | 权限码 | 日志 | 用途 |
|------|------|------|--------|------|------|
| POST | `/system/auth/login` | 公开 | - | 用户登录，排除 `password` | 后台用户名密码登录，返回 Token、用户信息和用户模型 |
| GET | `/system/auth/password-crypto` | 公开 | - | 否 | 获取密码 RSA-OAEP 加密公钥和一次性 nonce |
| POST | `/system/auth/refresh` | 登录 | - | 否 | 使用当前 Bearer Token 刷新访问令牌 |
| POST | `/system/auth/profile` | 登录 | - | 否 | 获取当前登录用户资料 |
| GET | `/system/auth/ui-meta` | 公开 | - | 否 | 获取界面元信息 |
| PUT | `/system/auth/profile` | 登录 | - | 更新个人资料 | 修改当前用户资料 |
| PUT | `/system/auth/preferences` | 登录 | - | 保存界面配置 | 保存当前用户界面偏好 |
| PUT | `/system/auth/password` | 登录 | - | 修改登录密码，排除密码字段 | 当前用户修改密码 |
| GET | `/system/auth/codes` | 登录 | - | 否 | 获取当前用户权限码 |
| POST | `/system/auth/logout` | 登录 | - | 用户登出 | 退出登录并清理登录态 |

## 使用说明

- 登录业务失败返回项目业务错误，不使用 `401`。
- `401` 只表示 Token 缺失、过期、无效或刷新失败。
- 密码、旧密码、新密码不会明文进入操作日志。
- 用户表中的密码由 `SystemUser::setPasswordAttribute()` 使用 `password_hash()` 写入哈希值；登录、改密、用户新增/重置等请求中的密码字段必须先用服务端公钥做 RSA-OAEP 加密，禁止明文字符串直接提交。
- 登录和资料接口返回 `auth_user_model=System\Model\SystemUser`，前端据此确认当前登录态属于后台入口。
- `/system/auth/ui-meta` 公开访问，只返回低敏界面展示配置，供登录页和应用启动初始化使用。

## 密码加密参数

`GET /system/auth/password-crypto?count=1`

该接口公开访问，用于在发送密码字段前获取当前服务端公钥和一次性随机数。RSA 私钥由后端自动维护在运行目录，部署人员不需要配置证书、公钥或私钥路径；如果 Swoole CLI、容器或服务器缺少系统 `openssl.cnf`，服务会在同一密钥目录自动维护内部最小 OpenSSL 配置，避免首次生成 RSA 私钥失败。

| 字段 | 说明 |
|------|------|
| `kid` | 当前密钥标识，由当前运行时公钥指纹生成，例如 `runtime-xxxxxxxxxxxxxxxx`；请求密码对象必须原样带回 |
| `alg` | 固定为 `RSA-OAEP` |
| `hash` | 固定为 `SHA-1`，与 PHP OpenSSL 的 OAEP 默认参数保持一致 |
| `public_key` | PEM 格式 RSA 公钥，前端使用 `spki` 导入 |
| `nonce_ttl` | nonce 有效期，默认 120 秒 |
| `nonces` | 一次性 nonce 数组，`count` 可按本次需要加密的密码字段数量申请 |

运行目录约束：

| 文件 | 说明 |
|------|------|
| `runtime/keys/password_crypto.pem` | 服务端自动生成的 RSA 私钥文件，只保存在服务端，不对外下发 |
| `runtime/keys/openssl.cnf` | 服务端自动生成的最小 OpenSSL 配置，仅用于缺少系统 `openssl.cnf` 的运行环境 |

以上文件均由服务端自动维护，部署和在线调试都不需要手动创建；生产环境只需要保证 `runtime/keys` 所在目录可写并纳入备份策略。

密码字段最终请求形态统一为：

```jsonc
{
  // kid: 来自 password-crypto 返回的密钥标识，服务端会校验是否为当前启用密钥。
  "kid": "runtime-key-id",
  // nonce: 来自 password-crypto 返回的单次随机数；每个 nonce 只能成功或失败地使用一次。
  "nonce": "server-issued-nonce",
  // ciphertext: RSA-OAEP(SHA-1) 加密后的 base64 密文。
  "ciphertext": "base64-rsa-oaep-ciphertext"
}
```

密文原文是 JSON 字符串：`{"v":1,"purpose":"...","nonce":"...","ts":1710000000,"value":"真实密码"}`。服务端会校验 `purpose`、`nonce`、时间戳和一次性消费状态，解密后再进入原有 `password_verify()` 或 `password_hash()` 流程。

## 登录请求

`POST /system/auth/login`

常见请求体：

```jsonc
{
  // username: 必填，登录用户名。服务端会去除首尾空格后查询账号。
  "username": "admin",
  // password: 必填，RSA-OAEP 加密对象；purpose 固定为 system.auth.login.password。
  "password": {
    // kid: password-crypto 返回的密钥标识。
    "kid": "runtime-key-id",
    // nonce: password-crypto 返回的一次性随机数，登录密码占用一个 nonce。
    "nonce": "server-issued-nonce",
    // ciphertext: 加密 {"v":1,"purpose":"system.auth.login.password","nonce":"...","ts":当前秒级时间戳,"value":"真实密码"} 后得到的 base64。
    "ciphertext": "base64-rsa-oaep-ciphertext"
  }
}
```

返回内容通常包括：

| 字段 | 说明 |
|------|------|
| `token` | 访问令牌，后续业务请求使用 `Authorization: Bearer <token>` |
| `user` | 当前用户基础信息，以及已加载的角色、部门、岗位关系 |

登录接口会记录操作日志，但 `password` 字段被排除。账号不存在、密码错误、账号禁用属于业务失败，不返回 `401`。

## Token 刷新

`POST /system/auth/refresh`

刷新接口不接收刷新令牌请求体，它会读取当前请求头中的 Bearer Token，并通过后端 Token 服务生成新 Token：

```jsonc
{
  // Authorization: 必填，当前仍可刷新的访问令牌。
  "Authorization": "Bearer <token>"
}
```

刷新成功后标准响应的 `data` 为新的 Token 字符串。Token 缺失、过期、无效或刷新失败时返回 `401`，前端应清理登录态并跳转登录页。

## 当前用户资料

`POST /system/auth/profile`

用于前端初始化用户信息。通常返回：

- 用户基础字段：用户名、昵称、头像、手机号、邮箱、签名、状态。
- 关系信息：角色、部门、岗位。
- 租户上下文：当前租户或平台相关信息。
- `auth_user_model`：固定为 `System\Model\SystemUser`。
- 前端展示所需扩展字段。

该接口要求登录，但不要求具体菜单权限。

## 更新个人资料

`PUT /system/auth/profile`

常见字段：

| 字段 | 说明 |
|------|------|
| `nickname` | 昵称 |
| `phone` | 手机号 |
| `email` | 邮箱 |
| `avatar` | 头像 URL |
| `signed` | 个性签名 |

该接口只允许修改当前登录人的个人资料，不应被用作管理员编辑任意用户的入口。管理员编辑用户应使用用户管理接口。

## 保存偏好

`PUT /system/auth/preferences`

用于保存当前用户界面偏好，例如主题、布局、表格偏好等。偏好属于用户态配置，不应存放密钥、Token、业务大对象或跨用户共享配置。

## 修改密码

`PUT /system/auth/password`

常见请求体：

```json
{
  "old_password": {
    "kid": "runtime-key-id",
    "nonce": "nonce-for-old-password",
    "ciphertext": "base64-rsa-oaep-ciphertext"
  },
  "new_password": {
    "kid": "runtime-key-id",
    "nonce": "nonce-for-new-password",
    "ciphertext": "base64-rsa-oaep-ciphertext"
  }
}
```

边界：

- 必须登录。
- 必须校验旧密码。
- 新密码写入前由服务端哈希。
- `old_password`、`new_password` 不进入操作日志。
- 修改成功后，前端可按安全策略要求用户重新登录。

## 权限码和退出

`GET /system/auth/codes` 返回当前用户拥有的权限码集合，用于前端按钮级控制。该接口结果来自角色和权限节点，不是安全边界；后端接口仍以 `#[Auth]` 为准。

`POST /system/auth/logout` 清理当前登录态并记录登出日志。退出后前端应清理本地 Token、用户信息、菜单和权限缓存。

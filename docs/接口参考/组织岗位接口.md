# 组织岗位接口

## 部门接口

控制器：`plugin/System/src/Controller/DeptController.php`
前缀：`/system/dept`

| 方法 | 路径 | 认证 | 权限码 | 日志 | 用途 |
|------|------|------|--------|------|------|
| GET | `/system/dept/index` | 权限 | `system.dept.index` | 否 | 部门列表 |
| GET | `/system/dept/recycle` | 权限 | `system.dept.index` | 否 | 部门回收站 |
| GET | `/system/dept/tree` | 权限 | `system.dept.index` | 否 | 部门树 |
| GET | `/system/dept/info/{id}` | 权限 | `system.dept.index` | 否 | 部门详情 |
| POST | `/system/dept/create` | 权限 | 默认部门创建权限 | 新增系统部门 | 新增部门 |
| PUT | `/system/dept/update/{id}` | 权限 | 默认部门编辑权限 | 编辑系统部门 | 编辑部门 |
| DELETE | `/system/dept/delete/{ids}` | 权限 | 默认部门删除权限 | 删除系统部门 | 删除到回收站 |
| DELETE | `/system/dept/real-delete/{ids}` | 权限 | 默认部门彻底删除权限 | 彻底删除系统部门 | 彻底删除 |
| PUT | `/system/dept/recovery/{ids}` | 权限 | 默认部门恢复权限 | 恢复系统部门 | 恢复 |
| PUT | `/system/dept/status/{id}` | 权限 | `system.dept.update` | 更新部门状态 | 启用/禁用 |
| PUT | `/system/dept/sort/{id}` | 权限 | `system.dept.update` | 更新部门排序 | 更新排序 |
| GET | `/system/dept/options` | 权限 | `system.dept.index` | 否 | 部门选项 |
| GET | `/system/dept/users/{id}` | 权限 | `system.dept.index` | 否 | 部门用户 |
| GET | `/system/dept/statistics` | 权限 | `system.dept.index` | 否 | 部门统计 |

### 部门查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `name` | string | 部门名称 |
| `leader` | string | 负责人 |
| `phone` | string | 联系电话 |
| `status` | number | 启用状态 |
| `page` / `pageSize` | number | 分页参数，树接口可能不使用分页 |

### 部门写入字段

| 字段 | 说明 |
|------|------|
| `pid` | 父级部门 ID，根节点通常为 `0` |
| `name` | 部门名称 |
| `leader` | 负责人 |
| `phone` | 联系电话 |
| `sort` | 排序 |
| `status` | 启用状态 |
| `remark` | 备注 |

### 部门树和数据范围

部门树是角色数据范围的重要基础：

- “本部门”依赖用户所属部门。
- “本部门及下级”依赖部门父子关系。
- “自定义部门”依赖明确的部门集合。

因此部门删除、恢复、移动和禁用都需要谨慎。删除前应确认没有子部门和用户占用；如果业务模块用部门字段做数据范围，也要确认该部门下是否存在业务数据。

### 部门用户

`GET /system/dept/users/{id}` 用于查看指定部门下的用户。该接口需要 `system.dept.index` 权限，并且结果仍应受当前登录人的数据范围影响。

### 岗位查询参数

| 参数 | 类型 | 说明 |
|------|------|------|

### 岗位写入字段

| 字段 | 说明 |
|------|------|
| `name` | 岗位名称 |
| `code` | 岗位编码，建议唯一且稳定 |
| `sort` | 排序 |

| `remark` | 备注 |

岗位主要用于用户身份标签和组织展示，不直接决定接口权限。需要权限控制时应使用角色；需要数据隔离时应使用部门数据范围。

## 选项和统计

| 接口 | 用途 | 注意事项 |
|------|------|----------|
| `/system/dept/options` | 部门下拉树 | 应受权限和数据范围约束 |
| `/system/post/options` | 岗位下拉选项 | 用于用户表单和筛选 |
| `/statistics` | 统计卡片 | 不能绕过数据范围 |

## 常见问题

- 部门树为空：检查 `system.dept.index` 权限、部门状态和数据范围。
- 删除部门失败：检查是否存在子部门、用户关联或业务数据依赖。
- 用户分配部门后数据范围不生效：重新登录并检查角色 scope 和 Mapper 归属字段。
- 岗位权限不生效：岗位不是权限来源，应检查用户角色和角色授权。

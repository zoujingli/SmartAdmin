## 使用说明

系统数据接口主要服务工作台、数据分析和系统配置页面。配置写入需要日志记录，但响应体不会完整进入日志。

## 工作台统计

`GET /system/data/index`

用于工作台首页的系统概览。常见内容包括：

- 用户、角色、菜单、部门、岗位等系统规模。
- 文件、日志、公告等运行数据概览。
- 最近操作、待办、系统能力摘要。

该接口需要 `system.data.index` 权限，适合授权给系统管理员或运营管理员。

## 系统信息

`GET /system/data/info`

用于查看运行环境和基础信息。可能包含：

| 类型 | 示例 |
|------|------|
| 应用信息 | 应用名称、版本、运行环境 |
| 运行时 | PHP、Swoole、Hyperf 版本 |
| 服务状态 | 数据库、缓存、上传驱动 |
| 系统能力 | 是否启用租户、发布快照、Phar 等 |

不要在该接口返回数据库密码、JWT 密钥、对象存储 Secret 等敏感配置。

## 系统配置

`GET /system/data/config` 读取可维护配置。
`PUT /system/data/config` 保存可维护配置。

适合放入系统配置的内容：

- 站点名称、Logo、主题等展示配置。
- 工作台展示开关。
- 低敏业务开关。
- 可由管理员维护的运行参数。

不适合放入系统配置的内容：

- 数据库密码、Redis 密码。
- JWT 密钥。
- 对象存储明文 Secret。
- 私钥、证书、第三方高权限 Token。
- 大体积业务正文或附件。

保存配置会记录操作日志，响应内容会在写入前统一脱敏并截断。

## 能力矩阵和待办

`GET /system/data/capabilities` 返回系统模块能力概览，适合用于文档、工作台或管理页展示。
`GET /system/data/todos` 返回工作台待办，适合提示管理员处理未完成事项。

二次开发新增模块后，如需要在工作台展示能力或待办，应通过 Service 聚合，不要让前端散落调用多个模块接口后自行拼装核心数据。

## 界面元信息

`GET /system/data/ui-meta`

登录态接口，用于返回系统界面需要的低敏元信息。它和公开的 `/system/auth/ui-meta` 都服务前端初始化，但调用场景不同：登录页和启动初始化优先调用 `/system/auth/ui-meta`；已登录后的兼容读取可调用 `/system/data/ui-meta`。

该接口的用户模型声明为 `UserModelInterface`，因此接受 `SystemUser`、`ProjectAccount` 等有效登录 Token。它不能返回业务数据、个人资料、权限码或高敏系统配置，也不能作为其他 `/system/**` 接口跨入口访问的依据。

## 系统参数

`GET /system/setting/info` 读取系统参数，主要用于系统参数页面和登录页展示配置。
`PUT /system/setting/info` 保存系统参数，写操作需要 `system.setting.save` 权限并记录操作日志。

系统参数面向可展示、可维护的低敏配置，例如系统名称、版本说明、登录页标题、Logo、版权和备案信息。不得把数据库密码、JWT 密钥、对象存储 Secret、第三方 Token 等高敏配置纳入该接口。

## 数据字典

`/system/dict` 提供通用字典分类和字典项管理。`pid = 0` 表示字典分类，子级表示字典项。分类 `code` 全局唯一；同一分类下字典项 `code` 和 `value` 应保持唯一。

常见字段：

| 字段 | 说明 |
|------|------|
| `pid` | 父级 ID，0 表示分类 |
| `code` | 字典分类或字典项编码 |
| `name` | 显示名称 |
| `value` | 字典项值，分类可为空 |
| `extra` | 扩展 JSON 配置 |
| `sort` | 排序值 |
| `status` | 状态，1 启用，0 禁用 |

`GET /system/dict/options?code=xxx` 只返回启用分类下的启用字典项，适合前端下拉、筛选和枚举展示。

## 清理缓存

`POST /system/data/clear-cache`

用于清理业务缓存。该接口需要 `system.data.clear-cache` 权限并记录操作日志。

使用场景：

- 菜单或权限同步后缓存未刷新。
- 系统配置调整后需要立即生效。
- 上传 runtime 配置或工作台统计缓存异常。

边界：

- 清理缓存不等于重启 Swoole。
- 修改 `.env`、Provider、依赖注入、注解等仍需要重启 Swoole 进程。
- 多实例环境要确保所有实例共享缓存，或对所有实例执行一致的清理策略。

## 前端关联

相关前端封装：

- `web/apps/web-antd/src/api/system/data.ts`
- `plugin/System/stc/view/data`（由 `plugin/System/plugin.json` 声明 view）
- 工作台页面和系统配置页面会间接使用这些接口；新增插件页面应优先放在自身 `plugin.view_root` 下。

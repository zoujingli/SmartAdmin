# 官网开放接口

官网开放接口用于官网前台、外部前端网关或第三方服务端读取 Website 插件公开数据。本文档只覆盖 `/website/**` 对外签名接口；后台资料维护、接口应用密钥、栏目内容和线索处理请查看 [官网管理接口](./官网管理接口.md)。

## 接口总览

开放接口必须使用 `appid + appkey + HMAC-SHA256` 签名，并按接口应用绑定的 `site_id + tenant_id` 过滤公开数据。面向普通访客的 `dmsai.cn` 官网页面不在后台实现，前端或第三方服务端需调用带签名的 `/website/**` 开放接口自行渲染。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/website/bootstrap` | 开放初始化，需签名 |
| GET | `/website/site` | 开放站点，需签名 |
| GET | `/website/nav/tree` | 开放导航树，需签名 |
| GET | `/website/channel/tree` | 开放栏目树，需签名 |
| GET | `/website/page` | 开放页面，需签名 |
| GET | `/website/content/index` | 开放内容列表，需签名 |
| GET | `/website/content/info/{id}` | 开放内容详情，需签名 |
| GET | `/website/content/slug/{slug}` | 开放 Slug 内容，需签名 |
| GET | `/website/block/list` | 开放区块列表，需签名 |
| POST | `/website/lead/create` | 开放线索提交，需签名 |

## 开放 API 签名规则

`/website/**` 只面向第三方服务端或可信服务端代理调用。调用方先在后台“官网管理 → 接口应用”创建应用，保存一次性返回的 `AppID` 与 `AppKey`。

所有开放 API 请求优先使用标准 `Authorization` 头传递签名认证信息：

| Header | 说明 |
|--------|------|
| `Authorization` | `Website-HMAC appid="...", timestamp="...", nonce="...", signature="..."` |

其中 `appid` 是接口应用 AppID；`timestamp` 是 Unix 秒级时间戳，允许与服务端相差 300 秒；`nonce` 是 8-64 位随机串，同一应用 300 秒内不可重复；`signature` 是 HMAC-SHA256 小写 hex 签名。历史 `X-Website-Appid`、`X-Website-Timestamp`、`X-Website-Nonce`、`X-Website-Sign` 头仍兼容，但新接入请使用 `Authorization`，降低 Nginx、网关和 CORS 配置复杂度。

签名串：

```text
METHOD + "\n" + PATH + "\n" + canonicalQuery + "\n" + sha256(rawBody) + "\n" + timestamp + "\n" + nonce
```

签名算法：

```text
sign = hash_hmac("sha256", stringToSign, appKey)
```

`canonicalQuery` 按参数名升序排序后使用 RFC3986 编码，POST JSON 请求体使用原始 body 计算 SHA256。验签通过后，服务端只使用接口应用绑定的站点；`site` 参数可以传应用绑定站点的 `code`、主域名或别名，但不能切换到其他站点。

接口权限按应用的 `scopes` 控制：`bootstrap` 会组合读取站点、导航、栏目和首页区块，需要 `site:read`、`nav:read`、`channel:read`、`block:read`；`page` 会组合读取站点、栏目/内容命中结果和页面区块，需要 `page:read`、`site:read`、`channel:read`、`content:read`、`block:read`；内容列表/详情、区块列表、线索提交分别使用 `content:read`、`block:read`、`lead:create`。

PHP 示例：

```php
$appId = 'your-app-id';
$appKey = 'your-app-key';
$method = 'GET';
$path = '/website/content/index';
$query = ['channel' => 'news', 'type' => 'news', 'page' => 1, 'pageSize' => 10];
ksort($query);
$rawBody = '';
$timestamp = (string)time();
$nonce = bin2hex(random_bytes(8));
$canonicalQuery = http_build_query($query, '', '&', PHP_QUERY_RFC3986);
$stringToSign = strtoupper($method) . "\n" . $path . "\n" . $canonicalQuery . "\n" . hash('sha256', $rawBody) . "\n" . $timestamp . "\n" . $nonce;
$sign = hash_hmac('sha256', $stringToSign, $appKey);
$headers = [
    'Authorization' => sprintf(
        'Website-HMAC appid="%s", timestamp="%s", nonce="%s", signature="%s"',
        $appId,
        $timestamp,
        $nonce,
        $sign
    ),
];
```

签名缺失、过期、重复或错误返回 body.code `401`；应用禁用、IP 不允许、scope 不允许或超限返回 body.code `403`；HTTP status 仍保持 `200`。

### 网关与浏览器调用说明

- 推荐由服务端、BFF、SSR 或同域 Nginx 代理完成签名调用，不要把 AppKey 放到普通浏览器页面。
- Nginx 反代到 Swoole 时建议显式保留 `Authorization`：`proxy_set_header Authorization $http_authorization;`。
- 如果确实需要跨域浏览器直连，CORS 只需允许 `Authorization` 与 `Content-Type` 等标准头；历史 `X-Website-*` 兼容头不建议作为新接入口径。

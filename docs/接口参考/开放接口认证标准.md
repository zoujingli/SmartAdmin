# 开放接口认证标准


## 接入流程

开放接口固定分为“换取 Token”和“业务调用”两段：

1. 后台在对应插件的“接口应用”页面创建应用，保存一次性返回的 `AppID/AppKey`。
2. 调用方使用 `appid/timestamp/nonce/sign` 调用插件认证入口换取 `access_token` 和 `refresh_token`。
3. 业务接口只携带 `Authorization: Bearer <access_token>`，不提交业务签名参数。
4. `access_token` 过期前可继续调用业务接口；过期后使用 `refresh_token` 调用 refresh 入口轮换新 Token。
5. 重置 AppKey、禁用应用、编辑导致 `token_version` 递增或 refresh 链路过期后，已签发 Token 立即失效。

`AppKey` 只用于 Token 换取阶段的本地签名计算，不能放入业务请求、URL query、前端页面或日志。

## 应用字段

开放接口应用表统一使用以下字段名：

| 字段 | 说明 |
|------|------|
| `appid` | 接口应用 AppID，开放接口公开身份标识 |
| `appkey` | 接口应用 AppKey 密文，明文只在创建或重置时返回一次 |
| `scopes` | 接口权限范围，由插件声明具体 scope |
| `ip_whitelist` | IP 白名单，空数组表示不限，支持单个 IP、`*` 和 IPv4 CIDR |
| `rate_limit` | 每分钟限流次数，`0` 表示不限流 |
| `access_token_ttl` | access_token 有效期秒数，范围 `300-86400` |
| `token_version` | Token 版本，重置密钥、禁用应用或后台切断凭证时递增 |
| `refresh_nonce` | refresh_token 单链路轮换随机串 |
| `refresh_expires_at` | refresh_token 链路过期时间 |
| `last_used_at` | 最近一次业务接口调用时间 |
| `last_used_ip` | 最近一次业务接口调用 IP |

界面展示使用 `AppID/AppKey`，接口字段和数据库字段固定使用 `appid/appkey/scopes`。

## 应用配置语义

| 配置 | 生效阶段 | 说明 |
|------|----------|------|
| `scopes` | 每次业务请求 | access_token 不冻结权限快照，后台调整后立即影响下一次业务请求 |
| `ip_whitelist` | 每次业务请求 | 空数组表示不限；非空时当前来源 IP 必须匹配规则 |
| `rate_limit` | 每次业务请求 | 以应用为维度做分钟限流，`0` 表示不限流 |
| `access_token_ttl` | Token 签发 | 返回到 `expires_in`，业务守卫也会检查 token 内的过期时间 |
| `token_version` | Token 签发与校验 | access/refresh token 内保存版本；数据库版本变化后旧 Token 失效 |
| `refresh_nonce` | refresh 校验 | refresh_token 单链路轮换凭据；刷新成功后旧 refresh_token 失效 |
| `refresh_expires_at` | refresh 校验 | refresh 链路最终过期时间，到期后必须重新用 AppKey 换取 Token |

## Token 换取

调用方先调用插件的认证入口换取 Token。请求参数固定为：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `appid` | string | 是 | 接口应用 AppID |
| `timestamp` | integer | 是 | Unix 秒级时间戳，允许与服务端相差 300 秒 |
| `nonce` | string | 是 | 16-64 位随机串，只允许字母、数字、下划线和中横线，同一应用 300 秒内不可重复 |
| `sign` | string | 是 | 小写 64 位十六进制 sha256 摘要 |

签名原文固定为：

```text
appid={appid}&nonce={nonce}&timestamp={timestamp}
```

签名算法固定为：

```php
hash('sha256', "appid={$appid}&nonce={$nonce}&timestamp={$timestamp}&appkey={$appkey}")
```

签名细节：

- `appid` 使用后台返回的 AppID 原值；当前生成的 AppID 均为小写。
- `timestamp` 使用 Unix 秒级时间戳，不使用毫秒。
- `nonce` 只允许字母、数字、下划线和中横线，长度 `16-64`。
- `sign` 使用 sha256 摘要结果的小写十六进制字符串，不使用 HMAC。
- Token 入口请求字段固定使用 `appid/timestamp/nonce/sign`。

PHP 签名示例：

```php
$appid = 'wapp_demo';
$appkey = 'plain-app-key';
$timestamp = time();
$nonce = bin2hex(random_bytes(8));
$raw = sprintf('appid=%s&nonce=%s&timestamp=%d', $appid, $nonce, $timestamp);
$sign = hash('sha256', $raw . '&appkey=' . $appkey);
```

Token 成功响应字段固定为：

| 字段 | 类型 | 说明 |
|------|------|------|
| `access_token` | string | 业务接口访问令牌 |
| `refresh_token` | string | 刷新令牌，刷新成功后旧值立即失效 |
| `token_type` | string | 固定 `Bearer` |
| `expires_in` | integer | access_token 有效期秒数 |
| `refresh_expires_in` | integer | refresh_token 有效期秒数，固定 30 天 |

## 业务调用

业务接口认证固定使用：

```http
Authorization: Bearer <access_token>
```

每次业务请求都会实时读取应用配置并校验：

- access_token 类型、过期时间、`appid`、`open_id` 和 `token_version`。
- 应用状态，禁用应用立即拒绝 access_token 与 refresh_token。
- 当前接口 scope。
- `ip_whitelist` 来源 IP。
- `rate_limit` 分钟限流。
- 插件自己的业务边界，例如 Website 的 `site_id + tenant_id`，Material 的 `tenant_id + region_ids`。

业务请求约束：

- `access_token` 只代表接口应用身份，不代表固定的数据权限快照。
- 后台禁用应用、重置密钥或调整 scope 后，下一次业务请求按数据库最新配置判断。
- `Authorization` 只放在 HTTP Header，不放在 query 或 body。
- 反向代理必须保留 `Authorization`、`X-Real-IP`、`X-Forwarded-For`，否则 Bearer 认证、IP 白名单和审计字段会失真。

## Refresh 轮换

`refresh_token` 使用单链路轮换。每次刷新成功都会生成新的 `refresh_nonce`，旧 refresh_token 随即失效。以下操作也会让已签发 Token 失效：

- 重置 AppKey。
- 禁用应用。
- 后台编辑导致 `token_version` 递增。
- refresh 链路到达 `refresh_expires_at`。

refresh 请求只提交 `refresh_token`。如果刷新失败，调用方必须重新走 Token 换取流程。

## 错误码

HTTP status 固定返回 `200`，业务状态读取响应体 `code`：

| code | 场景 |
|------|------|
| `401` | 缺少 Token、Token 无效或过期、签名错误、timestamp 超窗、nonce 重复、refresh_token 无效 |
| `403` | Token 有效但应用禁用、scope 不足、IP 不允许、限流、业务范围越权 |
| `500` | 参数或业务校验失败 |

## 联调检查清单

| 问题 | 检查点 |
|------|--------|
| token 入口返回 `401` | `timestamp` 是否为秒级、是否超过 300 秒；`nonce` 是否重复；签名原文顺序是否固定为 `appid/nonce/timestamp` |
| 业务接口返回 `401` | `Authorization` 是否被代理透传；access_token 是否过期；是否误把 refresh_token 用在业务接口 |
| 业务接口返回 `403` | 应用是否禁用；scope 是否包含当前接口；IP 白名单和分钟限流是否命中；业务范围是否越权 |
| refresh 返回 `401` | refresh_token 是否已被轮换使用过；应用是否重置密钥或禁用；refresh 链路是否到期 |
| 列表为空 | 插件业务范围是否正确，例如 Website 绑定站点、Material 授权区域和租户边界 |

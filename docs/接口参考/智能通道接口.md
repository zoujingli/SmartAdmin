# 智能通道接口

> **会员授权 / 不随开源版源码分发：** 本页为会员授权插件文档，公开文档站展示接口与使用说明，但开源仓不包含该插件源码或 ZIP，安装使用需会员授权或内部交付。

本页整理 `plugin/Smart` 中智能通道的后台接口。智能通道插件统一承载文本对话能力：管理员维护本空间智能通道接口配置，后台页面负责数据分析、配置、测试和日志统计，业务模块通过后端 `Plugin\Smart\Service\SmartGateway` 直接调用并取得统一结果。

## 接口内容

| 范围 | 说明 |
|---|---|
| 配置管理 | 维护 OpenAI Responses / OpenAI Compatible 文本对话配置，密钥加密保存，列表与详情只返回脱敏字段。 |
| 调度调用 | 未指定配置时按当前空间内启用配置的 `priority` 优先级选择；指定配置时必须属于当前空间且已启用。 |
| 测试与模板 | 后台提供支持标准说明、配置模板和连通性测试；不提供任意前端代理聊天接口。 |
| 调用统计 | 记录脱敏后的请求、响应、耗时、HTTP 状态和失败原因，并提供趋势、成功率、模型排行和配置排行。 |

## 数据分析返回

`/smart/analysis/overview` 只读取 `smart_record` 聚合结果，不返回请求与响应快照。支持 `created_at` 日期范围、`standard_code`、`config_id` 三类筛选；成功率、趋势和排行不接受前端传入的 `select`、`orderBy` 或 raw 表达式。

| 字段 | 类型 | 说明 |
|---|---|---|
| `summary` | object | 总调用、成功数、失败数、成功率、平均耗时、P90/P95 耗时、最慢耗时、今日调用和今日失败。 |
| `by_standard` | array | 按 OpenAI Responses / OpenAI Compatible 标准聚合的调用分布。 |
| `model_rank` | array | 按模型名称统计的 Top 10 调用排行。 |
| `config_rank` | array | 按智能通道配置 ID 聚合的 Top 10 调用排行，配置名优先展示当前配置名称。 |
| `trend` | array | 最近 7 天自然日趋势，缺少调用的日期会补 0。 |
| `recent_failures` | array | 最近 10 条失败记录的排障摘要，不包含请求响应快照。 |
| `slow_calls` | array | 当前筛选范围内耗时最高的 10 条调用摘要，不包含请求响应快照。 |

## 开放调用返回

`/smart/pool/status` 返回智能通道文本对话调用的最终可用性。默认未开放调用时 `call_enabled=false`，即便存在配置也不会启用；管理员开放调用后，还必须存在至少一条生效配置，最终 `enabled` 才会为 `true`。

| 字段 | 类型 | 说明 |
|---|---|---|
| `channel_code` | string | 首期固定 `chat`。 |
| `channel_text` | string | 调用类型名称，当前为文本对话。 |
| `call_enabled` | boolean | 是否已开放智能通道。 |
| `has_enabled_config` | boolean | 当前空间是否存在可用于真实调用的生效配置。 |
| `enabled` | boolean | 其它模块应使用的最终启用状态。 |
| `reason` | string | 不可用或已启用的展示说明。 |

## 配置字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | string | 是 | 配置名称，Service 会校验当前空间内唯一。 |
| `standard_code` | string | 是 | `openai_compatible` 或 `openai_responses`。 |
| `base_url` | string | 是 | 接口基础地址，例如 `https://api.openai.com/v1`。 |
| `api_path` | string | 否 | 为空时按标准使用 `/chat/completions` 或 `/responses`。 |
| `api_key` | string | 新增必填 | 接口密钥，加密保存；编辑时留空表示保留原密钥。 |
| `model` | string | 是 | 文本对话模型名称。 |
| `headers` | object | 否 | 附加请求头，`Authorization` 会被忽略，由系统按 API Key 生成。 |
| `options` | object | 否 | 默认模型参数，业务调用时可覆盖；Responses 路径只透传稳定参数。 |
| `timeout_ms` | integer | 否 | 请求超时时间，默认 `180000`，最小 `1000`。 |
| `priority` | integer | 否 | 调度优先级，数值越小越优先，默认 `100`。 |
| `status` | integer | 否 | `1` 启用，`0` 禁用。 |

## 调用与日志边界

- 业务模块不要直接读取 `smart_config.api_key`，应注入 `Plugin\Smart\Service\SmartGateway` 调用 `chat()`。
- 业务模块推荐使用 `SmartGateway::chat(array $payload): array` 作为标准入口；传参、返回字段和结构化输出示例见 [智能通道标准化调用](../开发指南/智能通道标准化调用.md)。
- 业务模块展示或执行智能通道调用前，统一调用 `Plugin\Smart\Service\SmartConfigService::isEnabled()` 判断；只有智能通道已开放且存在生效配置时才返回 `true`。
- `SmartGateway` 会在真实调用前再次检查开放调用状态，未开放时返回“智能通道未开放”；后台配置页的连通性测试不受开放调用状态限制，便于管理员先验证配置。
- OpenAI Compatible 使用 Chat Completions 形态：`model/messages/stream=false`，可透传 `max_tokens`、`temperature`、`top_p`、`response_format`、`tools` 等稳定字段。
- OpenAI Responses 使用 Responses 形态：`model/input/stream=false`，`max_tokens` 会映射为 `max_output_tokens`，`response_format.json_schema` 会映射到 `text.format`。
- 请求、响应、错误会写入 `smart_record`，其中 Authorization、API Key 等敏感字段会脱敏并截断。

## 备份与恢复边界

- 备份只包含当前空间 `smart_config` 配置记录，不包含调用日志 `smart_record`、菜单节点、统计计数、审计字段或其它系统参数。
- 备份文件固定使用 `.bin` 扩展名，内容为整体加密后的配置负载；API Key 会在加密包内保存，下载后的文件需按密钥文件管理。
- 恢复只会新增配置，不会替换当前空间已有配置；同名配置会自动追加 `（恢复）`、`（恢复2）` 等后缀生成新名称。备份里的主键、归属、成功/失败次数、最后调用时间和创建更新时间都会忽略。
- 恢复仍复用配置写入校验：只允许 `openai_compatible`、`openai_responses`，`headers.Authorization` 会被过滤，API Key 入库前重新加密。
- `.bin` 文件依赖当前系统密钥解密；跨环境恢复前需确认系统密钥一致，否则会提示备份文件解密失败。

# 模块文档总览

本章是 [系统功能](../系统功能/README.md) 和 [开发指南](../开发指南/README.md) 的补充专题，沉淀各业务模块的能力说明、权限清单、数据模型和前后端对接要点。

如果内容与系统功能或开发指南存在冲突，以主线文档和当前代码实现为准。

## 补充边界

模块文档只补充“某个模块怎么用、有哪些权限、涉及哪些表和页面”，不替代主线文档：

| 主线问题 | 应查看 |
|----------|--------|
| 平台能力是否存在、如何协作 | [系统功能](../系统功能/README.md) |
| 新模块或新插件如何开发 | [开发指南](../开发指南/README.md) |
| HTTP 字段和响应合同 | [接口参考](../接口参考/README.md) |
| 生产部署和升级命令 | [部署运维](../部署运维/README.md) |

## 文档编写约定

- 模块说明必须包含：接口、权限码、数据表/模型、关键流程
- 涉及升级影响时必须写“升级影响”与“回滚建议”
- 前端交互变化需同步补充页面与接口映射
- 会员授权或私有交付模块必须说明授权边界，避免把“文档可见”写成“开源版已包含源码”

# 文件管理与上传

## 模块定位 📁

本模块负责三件事：

1. 文件记录管理（`system_file`）
2. 上传通道配置治理（`config_upload`）
3. 上传运行时会话与分片流程（`system_file_upload`）

核心入口：`FileController`（路由前缀：`/system/file`）。

## API 能力总览 🧭

### 文件记录

- 列表：`GET /system/file/index`
- 回收站：`GET /system/file/recycle`
- 详情：`GET /system/file/info/{id}`
- 下载：`GET /system/file/download/{id}`
- 软删：`DELETE /system/file/delete/{ids}`
- 恢复：`PUT /system/file/recovery/{ids}`
- 彻底删除：`DELETE /system/file/real-delete/{ids}`
- 统计：`GET /system/file/statistics`
- 去重：`POST /system/file/dedupe`

### 上传配置

- 查看：`GET /system/file/upload-config`
- 更新：`PUT /system/file/upload-config`

### 上传运行时

- 运行配置：`GET /system/file/upload/runtime`
- 预检会话：`POST /system/file/upload/prepare`
- 中转上传：`POST /system/file/upload/relay`
- 中转分块：`POST /system/file/upload/relay-chunk`
- 分片签名：`POST /system/file/upload/part-sign`
- 完成上传：`POST /system/file/upload/complete`
- 中止上传：`POST /system/file/upload/abort`

## 上传链路（实现对齐）⚙️

`FileUploadService` 的核心流程：

1. `prepare`：校验文件元数据、后缀、MIME、大小，创建上传会话
2. 按驱动与文件大小选择传输方式：
   - 本地：中转单文件 / 中转分块
   - 云存储：直传单文件 / 直传分片 / 中转回退
3. 上传后执行真实元数据复核（大小、哈希、MIME）
4. 写入 `system_file`，返回统一文件结构

支持“秒传复用”：

- 哈希命名模式下可尝试按 hash 复用已有对象
- 仅允许复用当前数据范围内可见文件，避免猜 hash 越权复用

## 配置治理（UploadConfigService）🔐

配置特性：

- 默认值合并与结构校验
- 密钥字段加密存储（使用项目密钥）
- 后台展示时返回脱敏状态（`*_masked`、`*_configured`）
- 前端运行时只下发可执行规则，不下发密钥明文

支持驱动：

- `local`
- `alist`
- `oss`
- `qiniu`
- `cos`

## 文件删除与去重策略 🧹

`FileService` 在删除时不会盲删物理文件：

- 若同一对象仍被其他记录引用，跳过物理删除
- 仅在无其他引用时删除存储对象
- 去重支持单 hash 与批量模式，保留可用对象，删除冗余记录

这能避免“多记录引用同一对象”时出现误删。

## 安全与审计要求 🛡️

- 上传接口均受 `#[Auth]` 保护，配置修改与删除操作附带 `#[Logger]`
- 配置更新日志需排除密钥字段（`access_secret`、`secret_key` 等）
- 下载文件名经过净化处理，避免非法字符或路径注入
- 上传会话绑定创建者，禁止跨用户操作会话

## 运维检查清单 ✅

1. 上传配置保存后，密钥字段是否保持脱敏展示
2. 大文件上传是否按阈值切换到分块/分片流程
3. 上传完成后，文件记录 `preview_url/download_url` 是否可访问
4. 去重后是否只删除冗余记录且未误删共享对象
5. 日志目录与 SQL 日志目录是否按轮转策略落盘（见 `config/autoload/logger.php`）

## 对接建议 🤝

- 前端统一通过场景接口获取上传策略
- 新增存储驱动时同步补齐：
  - 驱动能力声明（是否支持直传/分片）
  - 配置校验规则
  - 密钥加密与日志脱敏字段

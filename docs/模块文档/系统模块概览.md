# 系统模块概览

> 本页只描述当前项目已落地的模块能力，聚焦 `plugin/System` 真实实现。🧭

## 1. 模块分层关系

- `Library`：提供基类、鉴权注解、日志注解、缓存与上下文能力
- `System`：核心后台模块（认证、用户、角色、菜单、部门、岗位、租户、文件、日志、公告、配置）
- `Builder`：构建与发布工具链

## 2. System 模块能力地图（按业务面）

### 🔐 认证与会话（Auth）

- 入口控制器：`AuthController`
- 关键服务：`UserService`、`MenuService`、`DataService`
- 典型接口：登录、刷新 token、用户资料、权限码、退出登录
- 运维关注：登录失败率、token 刷新异常、权限码返回是否正确

### 👥 用户与角色（User / Role）

- 入口控制器：`UserController`、`RoleController`
- 关键服务：`UserService`、`RoleService`、`AuthCacheService`
- 关键能力：用户资料维护、角色授权、节点分配、数据范围控制
- 运维关注：角色变更后权限缓存是否刷新、超管权限是否短路正确

### 🌲 菜单与组织（Menu / Dept / Post）

- 入口控制器：`MenuController`、`DeptController`、`PostController`
- 关键服务：`MenuService`、`DeptService`、`PostService`
- 关键能力：菜单树维护、组织结构维护、岗位基础数据维护
- 运维关注：菜单层级变更后路由是否正确、部门树结构是否出现循环

### 📁 文件与上传（File）

- 入口控制器：`FileController`
- 关键服务：`FileService`、`FileUploadService`、`UploadConfigService`
- 关键能力：文件记录、上传通道配置、本地/云上传会话
- 运维关注：上传配置安全（密钥脱敏）、回收站清理策略、重复文件治理

### 📣 公告通知（Notice）

- 入口控制器：`NoticeController`
- 关键服务：`NoticeService`
- 关键能力：公告发布、投放范围管理、通知收件与回收站
- 运维关注：通知投放准确性、公告状态与可见范围一致性

### 📊 数据看板与系统配置（Data）

- 入口控制器：`DataController`
- 关键服务：`DataService`、`OnlineUserService`
- 关键能力：系统统计、能力矩阵、在线会话、配置读写、缓存清理
- 运维关注：在线统计是否符合数据范围、全局缓存清理是否影响业务

### 🧾 日志审计（Logs）

- 入口控制器：`LogsActionController`、`LogsChangeController`
- 关键服务：`LogsActionService`、`LogsChangeService`
- 关键能力：操作日志写入、变更日志关联、统计分析、实时指标
- 运维关注：敏感字段脱敏、日志膨胀控制、归档任务是否正常

### 🏢 租户管理

- 入口控制器：`TenantController`
- 关键服务：`TenantService`
- 关键能力：租户资料、状态、有效期、租户管理员开通、租户统计
- 运维关注：租户状态和有效期是否影响登录，普通业务表是否正确接入 `tenant_id`

## 3. 同步与治理链路（命令层）

- `xadmin:menu:sync`：插件 `plugin.json` 菜单清单与 `system_menu` 对齐
- `xadmin:node:sync`：注解节点 + 菜单 code 与 `system_node` 对齐

插件清单、菜单和节点同步是权限治理的基线，建议纳入发布前检查。✅
